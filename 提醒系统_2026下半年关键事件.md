## 一、提醒系统架构

### 三层提醒体系

```
第一层：战略级提醒（季度）
  └─ 中期选举（11/3）、基准修订（12月）

第二层：战术级提醒（月度）
  └─ 每月非农（第一个周五）、FOMC会议

第三层：操作级提醒（周度）
  └─ CPI、PPI、PCE、初请、JOLTS
```

### 时间窗口设计

| 时间窗口 | 提醒内容 | 操作 |
|---------|---------|------|
| **T-7** | 事件预告 | 开始准备 |
| **T-3** | 关键提醒 | 完成准备 |
| **T-1** | 最终检查 | 制定预案 |
| **T+0** | 事件发生 | 实时记录 |
| **T+1** | 复盘 | 完成月度跟踪 |
| **T+7** | 中期检查 | 验证市场反应 |
| **T+30** | 长期验证 | 完成案例库 |

### 提醒方式

| 方式 | 工具 | 频率 |
|------|------|------|
| **Obsidian 标签** | #提醒 #[事件类型] | 手动 |
| **日历同步** | 可导出到 Google/Outlook 日历 | 手动 |
| **Cron 提醒** | Claude Code ScheduleWakeup | 自动 |

## 七、提醒系统使用指南

### Obsidian 集成方案

#### 方案1：标签筛选（推荐）

在所有事件笔记中使用统一标签：

```markdown
---
tags: [提醒, [事件类型], [月份]]
priority: [🚨极高/❌高/⚠️中]
---

# [事件标题]
```

然后用 Obsidian 的 **Search 功能** + 标签筛选快速查找。

#### 方案2：日历视图

把每个事件的关键日期记录在 Obsidian 日记（Daily Notes）中。

#### 方案3：Dataview 查询

如果安装了 Dataview 插件，可以用以下查询自动显示：

```dataview
TABLE priority, date, type
FROM ""
WHERE contains(tags, "提醒")
SORT date ASC
```

### Claude Code ScheduleWakeup 集成

可以为关键事件设置自动提醒：

```javascript
// 7月14日 CPI 提醒（提前7天 = 7月7日）
ScheduleWakeup({
  delaySeconds: 7 * 24 * 3600,  // 7天
  reason: "7天后7/14 CPI公布",
  prompt: "准备CPI事件追踪"
})
```

### 通知规则

| 事件类型 | T-X | 提醒方式 |
|---------|-----|---------|
| 🚨 极高（中期选举/重要FOMC） | T-7, T-3, T-1 | 多次提醒 |
| ❌ 高（重要数据） | T-3, T-1 | 2次提醒 |
| ⚠️ 中（常规数据） | T-1 | 1次提醒 |
