## 行为准则

### 我们的承诺

为了营造一个开放和包容的环境，作为贡献者和维护者的我们承诺：不论年龄、体型、残疾、族裔、性别特征、性别认同和表达、经验水平、教育水平、社会经济地位、国籍、个人形象、种族、宗教或性取向，任何人都能享受无骚扰的参与体验。

### 我们的准则

有助于创造积极环境的行为包括：
- ✅ 使用欢迎和包容的语言
- ✅ 尊重不同的观点和经验
- ✅ 优雅地接受建设性批评
- ✅ 关注对社区最有利的事情
- ✅ 对其他社区成员表示同理心

不当行为包括：
- ❌ 使用性化的语言或图像
- ❌ 挑衅、侮辱/贬损评论、个人或政治攻击
- ❌ 公开或私下骚扰
- ❌ 未经许可发布他人的私人信息

## 如何贡献

### 方式一：通过 GitHub（推荐）

#### 1. Fork 仓库

访问 https://github.com/d83636126-pixel/ai-finance-knowledge
点击右上角 **"Fork"** 按钮。

#### 2. Clone 到本地

```bash
git clone https://github.com/你的用户名/ai-finance-knowledge.git
cd ai-finance-knowledge
```

#### 3. 创建分支

```bash
git checkout -b feature/你的功能名
# 例如：git checkout -b feature/2026年7月非农分析
```

#### 4. 修改/添加笔记

按照 [笔记规范](#笔记规范) 进行修改。

#### 5. 提交更改

```bash
git add .
git commit -m "feat: 添加2026年7月非农分析"
git push origin feature/你的功能名
```

#### 6. 创建 Pull Request

在 GitHub 上点击 **"New Pull Request"**。

### 方式二：直接提交 Issue

如果你不想修改代码，但想：
- 🐛 报告错误
- 💡 提出建议
- ❓ 提问

直接在 GitHub 上创建 Issue。

## 笔记规范

### 命名规范

```
【类型】_主题_日期.md

类型：
- 方法论_xxx        —— 方法论文档
- 速查_xxx          —— 速查卡片
- 月度跟踪_xxx      —— 月度跟踪笔记
- 案例_NNN_xxx     —— 案例库笔记
- 深度分析_xxx      —— 深度分析
- 推演_xxx          —— 未来推演
- 事件_xxx          —— 事件追踪
- [YYYY]年[M]月xxx —— 月度数据
- 季度复盘_xxx      —— 季度复盘
- 提醒系统_xxx      —— 提醒系统
- 模拟投资组合_xxx   —— 模拟投资
```

### 必填头部

每个 Markdown 笔记开头应包含 YAML 头部：

```markdown
---
title: 笔记标题
tags: [标签1, 标签2, ...]
created: YYYY-MM-DD
updated: YYYY-MM-DD
type: 笔记类型
---

# 笔记标题

正文内容...
```

### 推荐标签

```
#非农 #CPI #FOMC #注水识别 #方法论
#美联储 #沃什 #鲍威尔 #特朗普 #关税
#通胀 #失业率 #时薪 #投资 #宏观
#案例库 #事件追踪 #月度跟踪 #季度复盘
```

### 双向链接

- 同主题笔记之间必须双向链接
- 使用 `[[笔记名]]` 格式
- 链接文件名而非完整路径

### Markdown 规范

- 使用 `##` 作为二级标题
- 使用表格组织结构化数据
- 使用引用块 (`>`) 标注重要观点
- 关键数据加粗 (`**`)

## Git 提交规范

### 提交信息格式

```
<类型>: <描述>

[可选] 详细说明
```

### 类型说明

| 类型 | 用途 | 示例 |
|------|------|------|
| `feat` | 新增笔记/功能 | `feat: 添加2026年7月非农分析` |
| `fix` | 修复错误 | `fix: 修正5月非农数据` |
| `docs` | 文档更新 | `docs: 更新README` |
| `style` | 格式调整 | `style: 优化笔记排版` |
| `refactor` | 重构 | `refactor: 重组方法论结构` |
| `test` | 测试相关 | `test: 验证注水识别准确度` |
| `chore` | 其他 | `chore: 更新.gitignore` |

### 示例

```bash
git commit -m "feat: 添加7月14日CPI事件预分析"
git commit -m "fix: 修正Q2季度复盘中的NFP数据"
git commit -m "docs: 更新README中的知识图谱导航"
```

## 方法论迭代流程

### 触发条件

| 触发 | 动作 |
|------|------|
| 预测误差 > 20% | 审查方法论相关步骤 |
| 连续2次命中同一红旗但误判 | 调整红旗阈值 |
| 新发现的有效红旗 | 增加新红旗项 |
| 失效的话术预测 | 调整话术库 |
| 新政治事件影响注水量 | 更新政治背景评估 |

### 迭代步骤

```
1. 案例实战 → 数据公布后套用方法论
2. T+30 验证 → 4-5周后回测
3. 方法论准确度评估 → 命中率统计
4. 未命中 → 识别原因 → 更新方法论
5. 下次案例验证更新
```

### 版本管理

- 方法论 v0.x：内部迭代
- 方法论 v1.0：稳定版本
- 方法论 v2.0：重大升级

## Pull Request 流程

### 提交前检查清单

- [ ] 笔记符合命名规范
- [ ] YAML 头部完整
- [ ] 至少包含2个相关标签
- [ ] 双向链接正确
- [ ] 没有破坏现有结构
- [ ] 数据来源标注完整
- [ ] 通过 Markdown 语法检查

### PR 标题格式

```
[类型] 简明描述

示例：
- [feat] 添加2026年7月非农分析
- [fix] 修正通胀数据
- [docs] 更新README
```

### PR 描述模板

```markdown
## 变更类型
- [ ] 新增笔记
- [ ] 修复错误
- [ ] 文档更新
- [ ] 方法论改进
- [ ] 其他

## 变更说明
简述本次修改的内容。

## 关联笔记
- [[相关笔记1]]
- [[相关笔记2]]

## 测试清单
- [ ] 链接测试
- [ ] 数据准确性
- [ ] 格式检查
```
