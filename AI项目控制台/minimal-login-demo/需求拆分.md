# minimal-login-demo：需求拆分

## 项目目标

建立一个最小登录 demo，用于验证登录页面、表单校验、登录状态、路由保护等基础能力。

## 第一版大需求

- 本地能稳定启动。
- 登录流程可演示。
- 页面状态清楚：未登录、登录中、登录成功、登录失败。
- 后续可作为其他项目的登录模块参考。

## 功能拆分

### 1. 项目基础

- [ ] 确认项目启动命令。
- [ ] 确认前端框架和路由结构。
- [ ] 补充 README。
- [ ] 保持 Lovable 连接项目的 Git 历史安全。

### 2. 登录页面

- [ ] 登录表单。
- [ ] 用户名 / 邮箱输入。
- [ ] 密码输入。
- [ ] 提交按钮。
- [ ] 加载状态。
- [ ] 错误提示。

### 3. 表单校验

- [ ] 必填校验。
- [ ] 邮箱格式校验。
- [ ] 密码长度校验。
- [ ] 错误信息显示。

### 4. 登录状态

- [ ] 登录成功后的状态保存。
- [ ] 登出功能。
- [ ] 刷新页面后的状态处理。
- [ ] 登录失败处理。

### 5. 路由保护

- [ ] 未登录访问受保护页面时跳转登录页。
- [ ] 已登录访问登录页时跳转主页。
- [ ] 添加一个最小 dashboard 页面。

### 6. 测试与验收

- [ ] 跑通 lint。
- [ ] 跑通 test。
- [ ] 跑通 build。
- [ ] 记录验收结果。

## 暂不做

- 暂不接真实支付。
- 暂不做复杂权限系统。
- 暂不做多租户。
- 暂不做完整后端系统，除非后续明确需要。

## 第一阶段验收

- [ ] 本地能启动。
- [ ] 能看到登录页面。
- [ ] 能完成一次模拟登录。
- [ ] 有失败提示。
- [ ] 有登出入口。
- [ ] 有 README 说明。
