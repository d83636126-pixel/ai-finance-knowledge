# 讨论：图谱可信度加强 × Obsidian 非农研究 × 不臃肿

## 已落地的小改动

- 应用内 Tab **「知识库」→「图谱目录」**（仅改显示名与文案；不重构 API）。

## 1. 现阶段原则：不臃肿

同意。当前只做三件事级的收口：

1. 命名澄清（已做）
2. **图谱链路可考证**（加强可信度，但用最小手段）
3. Obsidian 非农研究 → **选择性喂图谱**，禁止整库灌入

不做：新 CMS、双向同步引擎、大而全知识中台、为改名而大重构。

## 2. 「链路可考证」在现有条件下指什么

目标：现有边上的每一跳，都能回答 **凭什么存在 / 怎么验 / 错了怎么改**。

最小加强（建议按此顺序，仍保持精简）：

| 项 | 做法 | 不做什么 |
|----|------|----------|
| 边 provenance | 每条关键边补齐 `note` + 可选 `source`/`as_of`（文档或字段） | 不上新数据库 |
| 验证挂钩 | 边或路径能指向已有 verifier（有则挂，无则标「待验」） | 不为每条边新造数据源 |
| 健康检查 | 扩展现有 health：缺 note、缺 verifier、孤立传导 | 另做一套审计产品 |
| 盲测反哺 | V2.0 用命中/未命中标「该边存疑」 | 先上推送再验边 |

「现有条件下可考证」≠「每条边都有完美学术证明」；= **可追溯主张 + 可对照数据或明确「未验证」**。

现状一例：`nfp_release → chair_interpretation` 仅有一句 note「非农反映劳动力市场热度」，**没有**挂注水结构、修正值、私营/政府拆分——这正是 Obsidian 研究可以补、但应**精选**补的地方。

## 3. Obsidian 非农研究对图谱有没有用？

**有用，但是「原料」不是「整库粘贴」。**

### 高价值、可进图谱/Case 的（建议用）

| Obsidian 资产 | 对图谱的用途 |
|---------------|--------------|
| 注水红旗框架（政府占比、私营差距、修正等） | 可变成 **NFP 解读因子节点** 或 Case 检查清单，而不是只信 headline NFP |
| 「3 个月均值 vs 单月叙事」 | 约束事前/盲测输入：避免单月噪声直接驱动整条路径权重 |
| 月度复核结论（方向、是否注水） | **盲测/复盘标签**：给历史样本打 quality 标记，服务 V2.0 评分 |
| BLS 抓取与序列口径 | 已部分进工具/verifier；保持数据可复现 |

### 低价值、不要塞进图谱的（避免臃肿）

| 内容 | 原因 |
|------|------|
| 长篇月度叙事全文 | 图谱要的是结构边，不是文章 |
| 对话记录、操作手册全文 | 留在 Obsidian；图谱只链「结论要点」 |
| 一次性政治评论 | 除非沉淀成稳定因子 |

### 推荐接入方式（精简）

```text
Obsidian 方法论/案例
    → 提炼 3～7 条「可检验主张」
    → 写成边 note / Case 检查项 / 历史样本字段
    → 用 verifier 或事后窗口打分
    ✗ 不要把 vault 同步进「图谱目录」
```

结论：**非农研究是图谱在 NFP 这条线上的「质量校准器」**；用它加强 `nfp_release` 相关链路的可考证性，而不是把应用内目录改成第二 Obsidian。

## 4. 和 V2 / 核心动力的对齐

- 核心仍是图谱质量 + 验证（见 [[核心动力说明_2026-07-11]]）。
- V2.0 盲测可信度 = 系统化检验边；Obsidian 非农 = 优先把 **NFP 子图** 做可考证样板，再复制到 CPI 等。
- 「图谱目录」只负责找得到边，不负责生产可信度。
