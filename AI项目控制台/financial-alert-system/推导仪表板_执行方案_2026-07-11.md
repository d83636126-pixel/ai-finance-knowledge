# 推导仪表板 (Deduction Dashboard) — 执行方案

## 概述

**目标**：新增一个统一的「推导仪表板」标签页，将 5 个直观的可视化组件整合在一屏，让用户点击事件后即可一览传导全貌。

**原则**：
- 不修改引擎、不新增 API、不动已有功能
- 复用现有 ECharts (force/sankey/radar) 和 CSS 变量体系
- 与现有 17 标签页体系自然融合
- 单一新文件为主，HTML/现有 JS 做最小改动

| 文件 | 操作 | 行数 | 说明 |

## 架构设计

### 数据流

```
用户点击事件节点
  → toggleSelectEvent()
    → DeductionDashboard.render()  (如果仪表板可见)
      → fetchData() ─┬─ GET /api/propagation/paths?event=X
                      ├─ GET /api/propagation/exposure?event=X
                      └─ GET /api/propagation/derive?event=X
      → 浏览器端聚合数据
      → 渲染 5 个组件
```

3 个 API 并行调用，数据全部在浏览器端聚合，无需新后端。同一事件缓存避免重复请求（`_lastEventId` + `_cached*`）。

### 依赖关系

```
deduction_dashboard.js
  ├── 全局常量: TYPE_COLORS, DIR_COLORS, DIR_SYMBOLS, LINE_META, TYPE_LABELS, DIR_MAP
  │     (来自 propagation_app.js)
  ├── ECharts 全局: echarts
  │     (来自 static/echarts.min.js)
  └── CSS 变量: var(--surface), var(--border), var(--cyan), var(--green), var(--red), etc.
        (来自 static/style.css)
```

## 5 个可视化组件

### 1. 资产影响卡片 (Impact Summary Cards)

| 属性 | 值 |
|------|-----|
| 实现 | 2 列 CSS Grid (`.dd-cards-grid`) |
| 数据 | `/api/propagation/exposure` — `targets` |
| 卡片内容 | 方向箭头(↑↓)、置信度进度条(红→黄→绿)、传导权重、时滞、路径数量 |
| 排序 | 按 `target_priority` 升序（核心 target 优先） |

### 2. 多米诺传导链 (Domino Chain Diagram)

| 属性 | 值 |
|------|-----|
| 实现 | 复用 `.flow-chain` CSS（已在 propagation.html 定义） |
| 数据 | `/api/propagation/derive` — `chains[0].steps` |
| 选择策略 | 取 `composite_weight` 最高的链 |
| 展示 | `事件 → 中间节点 → … → 目标资产`，每步标注方向、权重、时滞、动态乘数 |

### 3. 资产影响雷达图 (Asset Impact Radar)

| 属性 | 值 |
|------|-----|
| 实现 | ECharts `radar` 类型，6 轴雷达 |
| 数据 | `/api/propagation/exposure` — `targets`（每资产一根折线） |
| 6 轴 | 传导权重、置信度、传导速度、路径数量、优先级、方向一致性 |
| 配色 | 6 色调色板循环（#00f8ff, #00ff88, #ffbb00, #ff00aa, #ff8800, #ff3355） |

### 4. 置信度仪表 (Confidence Gauge)

| 属性 | 值 |
|------|-----|
| 实现 | 水平渐变进度条（红→橙→黄→绿→绿），带 shimmer 动画 |
| 计算公式 | `overallConf = avgConf * 0.4 + derivedConf * 0.6` |
| 质量评级 | ≥80% 高置信度 / ≥60% 中等偏上 / ≥40% 中等 / ≥20% 偏低 / <20% 低置信度 |
| 附加统计 | 传导路径数、影响资产数、最大权重 |

### 5. 自然语言摘要 (Natural Language Summary)

| 属性 | 值 |
|------|-----|
| 实现 | 模板化中文生成（无需 LLM） |
| 数据 | `/api/propagation/derive` — `explanations` + `chains` + `counterfactuals` |
| 结构 | 事件名称 → 定性解释 → Top 3 传导链叙事 → 传导结论 + 置信度 + 时滞 → 反事实 |
| 示例 | `fed_rate_hike：美联储加息 → 紧缩货币政策信号。<br>1. fed_rate_hike → usd_index → btc_price ↑下跌<br>传导结论：↑下跌 · 置信度 75% · 预计 3 天内生效` |

## 集成细节

### propagation.html 改动

**① 侧边栏按钮**（在 `reason` 按钮后）：
```html
<button class="tab-btn" onclick="switchTab('deduction',this)"
  style="border-color:rgba(255,200,0,0.3);color:var(--yellow);">
  📊 仪表板
</button>
```

**② 新增面板**（在 `tabReason` 前）：
```html
<div class="card tab-panel" id="tabDeduction" style="flex:none;display:none;">
  <div class="card-title">📊 推导仪表板
    <span style="font-weight:400;color:var(--text-dim);">直观推导结果呈现 · 点击事件自动刷新</span>
  </div>
  <div id="deductionDashboard" style="max-height:660px;overflow-y:auto;">
    <div class="loading">← 选择一个事件节点查看仪表板</div>
  </div>
</div>
```

**③ 脚本引用**（在 `propagation_app.js` 后）：
```html
<script src="static/deduction_dashboard.js?v=1"></script>
```

### propagation_app.js 改动

**① TAB_IDS 加入 deduction**：
```js
var TAB_IDS = {
  // ... 原有 17 个 ...
  reason: 'tabReason',
  deduction: 'tabDeduction'
};
```

**② switchTab 增加分支**：
```js
if (name === 'deduction' && typeof DeductionDashboard !== 'undefined'
    && typeof DeductionDashboard.render === 'function')
  DeductionDashboard.render();
```

**③ toggleSelectEvent 增加自动刷新钩子**：
```js
var deductionTab = document.getElementById('tabDeduction');
if (deductionTab && deductionTab.style.display !== 'none'
    && typeof DeductionDashboard !== 'undefined'
    && typeof DeductionDashboard.render === 'function')
  DeductionDashboard.render();
```

## CSS 注入策略

`deduction_dashboard.js` 使用 **IIFE 自注入 CSS**：
- `document.createElement('style')` + `document.head.appendChild()`
- 所有样式以 `dd-` 前缀命名空间隔离（`.dd-container`, `.dd-card`, `.dd-gauge` 等）
- 与现有 `.flow-chain`、`.path-card` 等样式无缝复用
- 仅在首次加载时注入（`_styleId` 防重复）

## 空状态与错误处理

| 状态 | 显示 |
|------|------|
| 无选中事件 | `← 选择一个事件节点查看推导仪表板` |
| 加载中 | spinner + `加载仪表板数据...` |
| 数据加载失败 | `⚠ 数据加载失败，请重试` |
| 无传导路径 | `该事件未找到传导路径` |
| ECharts 未加载 | `ECharts 未加载`（雷达图区域降级） |

## 验证方法

1. 启动服务器：双击 `start_server.bat`
2. 打开 `http://127.0.0.1:8000/propagation.html`
3. `Ctrl+Shift+R` 硬刷新 → 确认进入**完整模式**
4. 点击事件节点（如 `fed_rate_hike`、`nfp_hot`、`cpi_hot`）
5. 切换到 **📊 仪表板** 标签页
6. 逐项检查：
   - [ ] **资产影响卡片**：2 列网格、方向箭头语义正确、置信度进度条着色合理、权重/时滞显示
   - [ ] **多米诺传导链**：横向流程图、节点颜色正确、连线有箭头动画
   - [ ] **自然语言摘要**：中文叙事通顺、事件名/传导链/反事实完整
   - [ ] **资产影响雷达图**：6 轴 ECharts 渲染、多资产折线区分、图例完整
   - [ ] **置信度仪表**：渐变条 + 百分比 + 质量评级 + 统计数据
7. 切换不同事件 → 仪表板自动刷新
8. 测试多种事件类型（NFP/CPI/FOMC/地缘）确保鲁棒性
9. 控制台无报错

## 性能考虑

| 措施 | 说明 |
|------|------|
| 并行 API 调用 | `Promise.all([paths, exposure, derive])` |
| 事件级缓存 | 同一事件不重复请求（`_lastEventId`） |
| 延迟雷达渲染 | `setTimeout(fn, 50)` 确保 DOM 就绪 |
| ResizeObserver | 雷达图自适应容器尺寸变化 |
| 清理机制 | `DeductionDashboard.cleanup()` 释放 ECharts 实例 |
