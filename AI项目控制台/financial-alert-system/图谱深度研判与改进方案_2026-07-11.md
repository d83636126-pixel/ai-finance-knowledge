# 图谱深度研判与改进方案

> 研判对象：`F:\financial-alert-system\propagation_graph.json`（124节点/287边/健康94分）
> 覆盖范围：架构、引擎、数据层、可视化、治理、V2.0盲测就绪度
> 改进原则：**最小必要手段，不臃肿**

## 一、总体评估

| 维度 | 评分 | 说明 |
|------|------|------|
| 架构设计 | ⭐⭐⭐⭐ | 三层传导逻辑清晰（事件→中间→目标），全球风险状态 hub 设计合理 |
| 图数据质量 | ⭐⭐⭐ | 表面指标健康，但深层存在结构缺陷 |

| V2.0 盲测就绪 | ⭐⭐ | Case 结构好，但核心数据节点严重脱节 |

## 二、三大致命结构缺陷

### 🔴 缺陷1：NFP/CPI/PPI 数据节点近乎孤立

```
当前状态：
  nfp_release ──[verifies,wt=0.8]──→ chair_interpretation   （仅1条边，死胡同）
  cpi_release ──[verifies]─────────→ chair_interpretation   （仅1条边，死胡同）
  ppi_release  （零边，完全孤立）
```

**后果**：
- Obsidian 中的 [[01_方法论/方法论_非农注水识别手册|非农注水识别框架]]（7项红旗、HHI、SAF、双概率模型）对图谱推理**完全无结构性影响**
- NFP 和 CPI 是市场最关键的两大数据发布，但在图谱中无法驱动任何传导
- chair_interpretation（主席解读）本身也没有任何出边——整条链路是双重死胡同
- V2.0 盲测时，NFP/CPI Case 只能返回"chair_interpretation 被验证"，而非"BTC/美元/美债方向是什么"

### 🔴 缺陷2：34个中间节点零入边（占55%）

关键悬空节点及其已有出边：

| 节点 | 出边→ | 缺失的上游 |
|------|-------|----------|
| `global_m2` | → liquidity, btc, eth, spx, gold（5条） | 全球央行政策 |
| `ois_spread` | → credit, liquidity, dollar_stress（3条） | Fed 利率决策 |
| `sofr_rate` | → treasury_yield（1条） | Fed 利率决策 |
| `yield_curve_2y10y` | → risk, vix, liquidity（3条） | Fed + 衰退信号 |
| `copper_price` | → breakeven, risk, growth（3条） | 中国/全球增长 |
| `china_credit_impulse` | → global_growth_expectation（1条） | pboc_easing |
| `china_property_stress` | → global_growth_expectation（1条） | 中国房地产事件 |
| `jgb_yield` | → yen_carry_unwind（1条） | boj_policy |
| `cross_currency_basis` | → global_dollar_stress（1条） | 美元压力源头 |
| `funding_rate` | → btc, eth, risk, liquidation（4条） | 加密市场状态 |
| `open_interest` | → btc_price（1条） | 加密市场状态 |
| `futures_basis` | → btc_price（1条） | 加密市场状态 |
| 七大科技股×7 | → nasdaq/spx | 什么也不驱动它们 |

**后果**：这些节点有"输出能力"但无"触发机制"，在图谱中是永远的旁观者。

### 🔴 缺陷3：global_risk_state 是单点推理瓶颈

```
8个输入 → global_risk_state → 5个target（btc/nasdaq/gold/usd/treasury）
```

- 这是全图最重要的 13 条边，决定了所有资产的方向判断
- **没有绕过 GRS 的替代路径**——所有事件必须经 GRS 才能到 target
- 若 GRS 权重/方向有偏，全部输出都会偏

## 五、改进方案

### 原则

1. **最小手段**：每加一个节点/边都回答"凭什么存在、怎么验证"
2. **先补传导链，再补 provenance**
3. **禁止整库灌入**：Obsidian 研究的结构化提炼，非全文同步
4. **每次变更后跑健康检查**
