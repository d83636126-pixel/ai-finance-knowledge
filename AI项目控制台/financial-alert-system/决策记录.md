### 决策（用户确认）

- **Vendor = Databento** 历史 API；不开通实时套餐。
- 资产映射：
  - Nasdaq → `GLBX.MDP3` / NQ 期货（parent `NQ.FUT` → 点时点活跃合约）
  - 美国 2Y → `GLBX.MDP3` / ZT 期货（parent `ZT.FUT`）
  - DXY → `IFUS.IMPACT` / DX 期货（parent `DX.FUT`）
  - Gold → 保留 Twelve `XAU/USD`
  - BTC → 保留 Binance
- 原因：单一接口覆盖 CME/CBOT + ICE Futures U.S.；CME DataMine 无法覆盖 ICE 的 DX；支持原始文件、元数据、成本预估，适合留证。

### 工程约束（已落地骨架）

1. 独立适配器：`scripts/lib/databento_*.js` + `scripts/nfp_fetch_databento_windows.js`（**不覆盖** Twelve 脚本）。
2. 拉取 `ohlcv-1m`，本地聚合成 5m。
3. **禁止**连续合约；冻结 raw symbol / instrument_id / 到期日 / 选约规则 / 原始 SHA-256。
4. 原始 `DBN.ZST`、请求参数、下载日志、转换结果留存于 `data/nfp_research/databento/<event>/`。
5. ZT 方向协议事前冻结：`artifacts/nfp_zt_direction_protocol_v1.json`（`futures_price_direction` + `yield_equivalent_direction`）。
6. 下载前必须 `metadata.get_cost`；需 `--confirm-purchase`；默认 `--max-cost 5`。
7. NQ+ZT+DX 五窗齐 + gold/btc 后才 `golden_candidate=true`；PILOT 卡不自动升黄金。
