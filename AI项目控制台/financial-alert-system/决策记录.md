## 2026-07-10：突发事件抓取策略

### 决策

- RSS：**中英文源都要**。
- 工作流：**先手动**——inbox 展示候选，用户点「分析」再跑路径报告。
- LLM：**要接**，但定位为 **归类助手**（建议 graph node + 摘要），不是自动改图谱或自动出结论；用户确认后才进入 PropagationEngine。

### 原因

关键词/rule 层负责廉价过滤与审计；LLM 补语义理解（中英文标题歧义、新表述）；图谱引擎与 evidence 层保持确定性可验证。避免 LLM 全自动导致误映射污染决策资产。

## 2026-07-05：确认图谱中心目标

### 决策

`financial-alert-system` 的图谱中心目标采用：

```text
中心目标 = 全球风险状态
输出目标 = 资产篮子
重点资产 = BTC + Nasdaq + Gold + USD + Treasury
```

### 原因

如果直接以 BTC 或某一个资产为中心，图谱会变成单资产解释器，全球宏观节点会被压缩成附属变量。

以“全球风险状态”为中心，可以同时解释：

- 美元压力
- 利率压力
- 流动性状态
- 风险偏好
- 地缘冲击
- 多资产反应

### 影响

- BTC 仍然是重要输出资产，但不是唯一中心。
- Nasdaq、Gold、USD、Treasury 进入第一版核心输出资产篮子。
- 后续补节点时，应优先补能影响全球风险状态和核心资产篮子的节点。
- 后续边和权重应围绕“事件 -> 状态 -> 资产”的链路设计。
