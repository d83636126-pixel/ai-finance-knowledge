# financial-alert-system：第二大版本（V2）需求大纲

## 定位

**V1** = 宏观传导研究闭环（讲清楚、验清楚、能存档）。  
**V2** = **以盲测可信度为主线**，再接「突发新闻 → 推送 → 影响分析 → 后续走向预判」。

一句话：

> 先证明图谱在历史盲测上靠谱；再把突发新闻变成可推送、可分析、可复盘的研究流。

## 成功标准（用户确认）

1. **盲测可信度高**：批量历史盲测可量化；路径/方向命中可统计；无数据穿越。
2. **突发新闻闭环**：新闻进入系统 → 推送提醒 → 自动/半自动分析影响 → 给出后续走向预判（情景，非点位承诺）。

## 主命题与辅线

| 线 | 角色 | 内容 |
|----|------|------|
| **A 主线** | 必须先硬 | 历史样本加厚、批量盲测、评分看板、防穿越 hardening |
| **B 辅线** | 主线达标后接入 | 突发扫描加强、推送（原 P3）、影响分析模板、走向预判情景 |
| **不做** | 边界 | 自动交易、点位/收益率承诺、云端多租户 SaaS |

## 分期建议

### V2.0 — 盲测可信度（A）

**目标**：能回答「这套图谱在历史上事前猜对了多少」。

- [ ] NFP/CPI 样本继续加厚（更早年份）；评估 PPI/FOMC 第一批
- [ ] **批量盲测**：一键跑一批事件 → 产出汇总表（方向命中、路径命中、资产篮子命中）
- [ ] **评分口径**（先定规则再写代码）：
  - 方向：偏热/偏冷/符合预期 vs 实际 surprise 方向
  - 路径：主路径节点是否在事后 evidence 中被激活
  - 资产：BTC/Nasdaq/Gold/USD/Treasury 方向是否与事后窗口一致（1D/3D/7D）
- [ ] 盲测报告强制 `as_of_cutoff` + `graph_version`；批量跑也禁止泄漏
- [ ] 可信度看板：按事件类型、年份、图谱版本切片
- [ ] 结果可写入存档 + Obsidian `reports/`（复用 V1 存档）

**V2.0 验收**

- 至少一类事件（建议 NFP）完成 ≥20 条可跑盲测样本，或连续 ≥12 个月样本。
- 批量盲测无穿越（自动校验全绿）。
- 看板能给出可解释的命中率，而不是单条手工感觉。

### V2.1 — 突发新闻分析流（B 前半）

**目标**：突发进来就能「归类 → 影响分析 → 预判情景」。

- [ ] Inbox/RSS/症状扫描稳定化；突发置信度与建议节点质量提升
- [ ] **影响分析模板**：冲击源 → 传导路径 → 资产篮子敞口 → 验证清单
- [ ] **后续走向预判**：输出 2～3 个情景（基准/上行/下行）+ 触发条件 + 观察指标；明确标注「情景预判，非预测承诺」
- [ ] 预判事后可复盘：到点后自动/手动对照实际走向，反哺可信度

**V2.1 验收**

- 一条示例突发：确认节点 → 生成影响分析 + 三情景预判 → 写入分析结果/Obsidian。
- 预判字段可在事后对比中打分（情景是否被触发）。

### V2.2 — 推送闭环（B 后半，原 P3）

**目标**：重要突发/日历事件能推到人，并带分析摘要。

- [ ] 推送通道（如 ServerChan / 企业微信 / 邮件，择一先做）
- [ ] 触发规则：日历临近、Inbox 高优先级、盲测/预判关键变更
- [ ] 推送内容：标题 + 影响摘要 + 主路径 + 预判情景一句话 + 打开链接
- [ ] 可选：轻量 Cron / 定时扫描（本机任务即可，不强制 7×24 云端）

**V2.2 验收**

- 人为制造一条高优先级 Inbox → 收到推送 → 点开可看完整分析。
- 推送失败有明确日志与重试，不静默丢。

## 「预判」口径（强制）

| 允许 | 禁止 |
|------|------|
| 情景分支、触发条件、观察窗口 | 「必涨/必跌」点位 |
| 基于图谱路径的条件概率表述 | 承诺准确率或收益 |
| 事后用同一套口径打分 | 用事后行情训练成「事前」话术 |

预判质量最终仍回到 **A 线盲测/复盘分数**，避免「会写报告但不可信」。

## 与 V1 的关系

- V1 能力全部保留：图谱、日历/Inbox、Case、Alert 桥接、结果面板、存档同步、Obsidian 笔记。
- V2 不推翻本地 BFF 架构；推送与批量盲测优先挂在现有 `local_server`。
- 「清空全部结果」等小体验项可夹在 V2.0 边角，不单独占主线。

## 知识库在 V2 中的位置

> 原始作用已澄清：[[AI项目控制台/financial-alert-system/知识库说明_2026-07-11]]  
> **显示名已改为「图谱目录」**（避免与 Obsidian 混淆）。  
> **结论：= 图谱可搜索只读目录，不是研究笔记库。**

V2 是否扩展该 Tab = **改定位**，须另决策；**当前不默认改造**。盲测/突发产出优先走「分析结果」与看板。

原则：[[AI项目控制台/financial-alert-system/讨论_图谱可信度与非农研究_2026-07-11]] — **现阶段不臃肿**；图谱链路可考证用最小手段；Obsidian 非农研究精选喂边/Case，不整库灌入。

## 建议实施顺序

1. 定盲测评分口径（文档 + 样例表）→ 再写批量跑分
2. 加厚 NFP（或 CPI）样本到可统计量级
3. 可信度看板最小版
4. 突发影响分析 + 三情景预判模板
5. 推送通道 + 触发规则
6. 用真实突发跑通「推送 → 分析 → 事后打分」全链路

2. 推送通道优先选哪个？
3. 预判默认观察窗：1D / 3D / 7D 哪个做主窗？
