# 数据佐证层第一版

## 背景

用户指出：当前图谱主要是理论推导，缺少真实数据佐证。

本轮开始建立“数据佐证层”，目标是让图谱报告能显示：

```text
这个验证指标有没有数据？
数据来源是什么？
当前状态是确认、部分确认、失效，还是暂无数据？
```

## 本次新增文件

```text
verifier_data_dictionary.json
market_snapshot.json
```

### verifier_data_dictionary.json

定义每个 verifier 对应的数据来源、symbol、单位和确认逻辑。

第一批覆盖：

```text
tips_10y_real_yield      FRED / DFII10
dxy_momentum             DXY
move_index               MOVE
acm_term_premium         NY Fed ACM
treasury_auction_tail    US Treasury auction result
btc_etf_flow_verifier    BTC ETF net flow
gold_etf_flow            Gold ETF flow / GLD holdings
nasdaq_breadth           Nasdaq breadth
```

### market_snapshot.json

保存当前市场快照。

第一版采用手动模式：

```text
mode: manual
```

每个 verifier 先有数据槽位：

```text
status
current_value
previous_value
change
direction
as_of
note
```

状态枚举：

```text
confirmed
partial
invalidated
no_data
```

## 本次代码改动

已更新：

```text
static/propagation_engine.js
propagation.html
```

### 引擎侧

- 加载 `market_snapshot.json`。
- 加载 `verifier_data_dictionary.json`。
- 每个 validation point 自动携带：

```text
evidence_status
evidence_label
data_source
data_symbol
data_unit
current_value
previous_value
change
actual_direction
as_of
evidence_note
```

### 页面侧

验证指标现在显示：

- 数据状态：暂无数据 / 已确认 / 部分确认 / 已失效。
- 数据来源。
- 数据 symbol。
- 当前值。
- 变化。
- 更新时间。

## 当前能力

现在系统已经从：

```text
纯理论推导
```

推进到：

```text
理论推导 + 数据槽位 + 证据状态显示
```

但目前还没有自动拉取实时数据。
