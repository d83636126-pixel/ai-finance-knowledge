# 核心 Target 输出排序

## 背景

前一轮已经给 target 节点建立了两层结构：

- 核心 target：BTC、Nasdaq、Gold、USD、Treasury
- 扩展 target：ETH、SPX、Altcoin Season

但这一步之前，分层主要体现在图谱节点和逆向归因下拉框中。路径报告、Case 报告、敞口汇总还没有稳定地把核心 target 放在优先阅读位置。

## 本次实现

已更新：

- `static/propagation_engine.js`
- `propagation.html`

实现内容：

- `findPaths()` 返回路径时携带 `target_tier` 和 `target_priority`。
- `getExposure()` 的 target 汇总携带 target 分层信息。
- `getPathReport()` 的资产敞口按 `target_priority` 优先排序。
- `getCaseReport()` 的资产敞口按核心 target 优先排序。
- Case 摘要增加核心观察目标说明。
- 路径报告摘要从“目标资产”改为“核心目标”口径。
- 页面中的资产敞口卡片显示“核心 target / 扩展 target”标签。
- 多事件合并敞口也按核心 target 优先展示。

## 设计原则

不改变底层路径搜索的强弱判断。

路径本身仍然按传导权重衡量强弱；改变的是报告阅读顺序，让系统先回答：

```text
这件事对 BTC / Nasdaq / Gold / USD / Treasury 的影响是什么？
```

再看扩展资产。
