# 数据源可信度与结构夯实（2026-07-13）

## 原则（本轮边界）

- **不做**券商对接 / 下单。
- **先做**整体结构：完备度 ≠ 可信度；代理不得伪装成主事实。
- 图谱边 `source_url` 仍是**标注**，不是实时验证（与研究台四层轴一致）。

## 结构：两条正交门禁

| 门禁 | 回答的问题 | 模块 |
|---|---|---|
| `data_quality`（完备度） | 字段齐不齐、鲜不鲜、OI 单位对不对 | `micro_layer.assessQuality` |
| `source_trust`（可信度） | 来源档次、跨所是否打架、熔断、代理是否主导 | `micro_source_trust.assessSourceTrust` |

任一硬门禁失败 → 决策约束 `data_quality_below_threshold` / `source_trust_below_threshold` → **弃权**。

### 可信度硬门禁（不可被其他项平均掉）

1. **多所覆盖** `multi_venue_coverage`（默认 ≥2）
2. **跨所 funding 日化价差** `funding_cross_check`（默认 spread ≤ 0.04）
3. **熔断未打开** `no_circuit_open`（`SourceHealth`）

### 来源档次（高→低）

`primary_exchange` → `secondary_exchange` → `aggregator` → `proxy` → `assumed` → `missing_paid` / `circuit_open`

诚实缺口（付费未接）：清算热图、真 MVRV/LTH —— 记入 `paid_gaps`，不填假数。
