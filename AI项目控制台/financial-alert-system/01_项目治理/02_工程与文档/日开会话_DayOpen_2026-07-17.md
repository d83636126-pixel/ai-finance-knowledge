# 日开会话（Day Open）· 使用方法

> 目的：每天打开本机项目时，自动把 `SUPABASE_URL` + `SUPABASE_ANON_KEY` 写入 `.env`，并启动 `local_server`（可 LAN：`HOST=0.0.0.0` + `FS_ACCESS_TOKEN`）。  
> 本机**不跑** 5 分钟抓取；抓取在 GitHub Actions → Supabase。

## 一句话流程

1. **每台电脑首次一次**：安装凭据  
2. **每天一次**：日开脚本  
3. 浏览器打开 `http://127.0.0.1:8000/`

---

## 重要：用 PowerShell 时怎么敲

Cursor / Windows 终端经常是 **PowerShell**，不要用 CMD 写法 `cd /d`。

```powershell
Set-Location F:\financial-alert-system
```

若 `npm` 报「禁止运行脚本」，不要先改系统策略，直接用 Bypass 调脚本（见下文）。

---

## 本机首次（每台机器只做一次）

密钥进 **Windows 凭据管理器**（当前用户），**不要写进本笔记正文**。

```powershell
Set-Location F:\financial-alert-system
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\install_day_open_secrets.ps1
```

按提示填写：

| 项 | 填什么 |
|---|---|
| `SUPABASE_URL` | `https://xxxx.supabase.co`（**不要**带 `/rest/v1/`） |
| `SUPABASE_ANON_KEY` | publishable / anon（**禁止** `sb_secret` / service_role） |
| `HOST` | 直接回车 → 默认 `0.0.0.0`（局域网可访问） |
| `PORT` | 直接回车 → 默认 `8000` |
| 生成 `FS_ACCESS_TOKEN` | 建议输入 `Y` |

等价 npm（若 execution policy 允许）：

```bat
npm run day:open:install-secrets
```

## 每天打开（日开）

### 方式 A：Obsidian 一键

双击或 Shell Commands：

```text
AI项目控制台\financial-alert-system\_sync\day_open.bat
```

> bat 内默认仓库路径：`F:\financial-alert-system`。换盘符/路径时改 bat 里的 `REPO=`。

### 方式 B：PowerShell（推荐，避开 npm 策略问题）

```powershell
Set-Location F:\financial-alert-system
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\day_open_dev_session.ps1
```

强制重写 `.env` 并重启服务：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\day_open_dev_session.ps1 -Force
```

### 方式 C：npm

```bat
npm run day:open
npm run day:open:force
```

成功标志：终端出现 `.env updated` / `local_server started`（或已在跑则 skip）。

## 打开页面

| 场景 | 地址 |
|---|---|
| 本机 | http://127.0.0.1:8000/ |
| 局域网其他设备 | http://\<本机IP\>:8000/ |

打开后会自动 remote-cache pull（**恢复快照**口径，不可当作本机实时签署）。

统一工作台「系统健康」可看异地采集 run 摘要。

## 换机器要不要同样操作？

**要。** 凭据在「这台 Windows 用户」里，不会随 Git / Obsidian 笔记自动带走。

新机器清单：

1. 拉齐代码仓（或拷贝），确认有 `scripts\day_open_dev_session.ps1`
2. 安装 Node
3. 再跑一遍「本机首次」`install_day_open_secrets.ps1`
4. 再跑「每天」`day_open_dev_session.ps1`
5. 若用 Obsidian bat：改 `_sync\day_open.bat` 的 `REPO=` 指向新机器上的仓库路径

同一套 Supabase 项目：URL + anon 可相同；`FS_ACCESS_TOKEN` 每台可各自生成。

## 可选：开机自动日开

```powershell
Set-Location F:\financial-alert-system
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\install_day_open_logon_task.ps1
```

- 任务名：`FinancialAlert-DayOpen`
- 登录后跑一次；脚本按**自然日**去重
- 卸载：`npm run day:open:remove-logon` 或同脚本加 `-Remove`

## 安全边界（必读）

| 项 | 规则 |
|---|---|
| anon / publishable | 可进凭据 + `.env` + 浏览器 public-config |
| service_role / `sb_secret` | **只** GitHub Actions Secrets；禁止 day-open / Obsidian / 浏览器 |
| Obsidian 笔记 | 只写步骤，**不写密钥明文** |
| 本机定时抓取 | 已停用；勿再装 `FinancialAlert-MicroCollect-5m` |

## 常见问题

### `cd /d` 报错

PowerShell 不认 `/d`。用：

```powershell
Set-Location F:\financial-alert-system
```

### `npm.ps1` 禁止运行脚本

改用 Bypass 直接跑 `.ps1`（见上文），或：

```powershell
cmd /c "cd /d F:\financial-alert-system && npm run day:open"
```

### Missing credential `FinancialAlertSystem/DayOpenSession`

这台机器还没做过首次安装 → 跑 `install_day_open_secrets.ps1`。

### 健康检查失败

看仓库 `artifacts\local_server_day_open.out.log` / `.err.log`。  
确认 Node 在 PATH；端口 8000 未被占用。

### 局域网打不开

确认 `HOST=0.0.0.0`、防火墙放行 8000、用本机局域网 IP（不是 127.0.0.1）。
