# 决策：远程采集面（Supabase）替代本机 5 分钟抓取

## 背景

本机 Windows 计划任务 `FinancialAlert-MicroCollect-5m` 每 5 分钟跑微观采集，导致电脑卡顿。用户要求：**继续自动抓取，但不在本机后台跑**。

用户提案：抓取程序放 Lovable + 数据存 Supabase，本机只访问网上库。

## 决策（2026-07-17 用户确认「可以」）

**采纳「远程写库 / 本机只读」数据面架构**，并作如下落地修正：

| 项 | 决定 |
|---|---|
| 数据库 | **Supabase Postgres** |
| 定时抓取宿主 | **优先** 小 VPS / GitHub Actions / Supabase Cron；**不**把 Lovable 当作唯一定时爬虫 |
| Lovable | 可选：**只读看数面板**（只用 anon key） |
| 本机 | 删除 5 分钟与 8 小时采集计划任务；用 `npm run supabase:pull` 拉库 |
| 密钥 | `service_role` 仅远程服务端；本机/Lovable 前端只用 `anon` |
| 与整改口径 | 这是**数据面异地化**，**不等于**产品上云 / `cloud_gate=pass` |

## 你需要完成的人工步骤（云账号）

1. 新建 Supabase 项目  
2. 跑迁移 SQL  
3. 把 `SUPABASE_URL` + `SUPABASE_ANON_KEY` 写入本机 `.env`  
4. 选一台**非本机**主机放 `SERVICE_ROLE` 并定时跑采集+push  
5. （可选）Lovable 接 anon 只读表做面板  

工程说明：代码仓 `supabase/README.md`
