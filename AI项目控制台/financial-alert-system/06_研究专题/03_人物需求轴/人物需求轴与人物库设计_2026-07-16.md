## 一、用户原始愿景（会话原述压缩）

### 宏观层

基于 NFP/CPI 等即时与历史数据，以及联储、政府相关人物发言，理清美政府经济—政治脉络；对现行与未来政策做**有数据依据**的推导；看清中长期大致走势。

### 微观层

在宏观推导出的中长期情景下，结合链上、量价、相关政策与加密圈发展脉络，对短中长期做**可数据证实**的判断；判断足够成熟、可靠后，再辅助交易。

### 与当前产品对照（结论）

| 目标块 | 产品现状 | 同向？ |
|--------|----------|--------|
| NFP/CPI 数据推导 | Case、日历、verifier、NFP 月度跟踪、盲测 as-of 规划 | 同向 |
| 人物发言 → 政策脉络 | 沃什框架 + Inbox 突发（人确认） | 弱同向 |
| 中长期走势 | 事件传导 / 风险篮子情景 | 部分同向 |
| 链上 + 量价 + 加密脉络 | 微观层偏 CEX 量价；链上/政策薄 | 量价同向，脉络不足 |
| 成熟后辅助交易 | 门禁 + 只观察；明示不下单 | 方向保留，阶段延后 |

**总判：** 方向大体相同，完成态不同。产品偏「事件传导工程 + 加密微观门禁」；愿景偏「政策脉络 OS → 情景 → 微观证实 → 交易」。中间缺口：人物/发言系统化、宏微事件主键、交易成熟度硬里程碑。

## 二、人物—发言—立场主库（要建什么）

不是新闻堆，是三层：

| 层 | 主键逻辑 | 用途 |
|----|----------|------|
| Person | `person_id` | 身份、机构、COI、关系边 |
| Speech | `speech_id` | 日期、URL、信源 A/B、议题、摘录 |
| StanceSnapshot | `person_id + topic + as_of` | 立场随时间 + 变化分 |

**分期：**

1. **MVP-0（2–3 周）**：人工主库，名单 ≤40，统一卡片，台账节奏  
2. **MVP-1**：Inbox/RSS 命中人名 → 人确认入库  
3. **MVP-2**：接到传导图 / NFP Case 主键  

**冲刺定位：** 对愿景是核心脊梁；对本月 NFP/微观观察是「并行下一核心」，不抢主线、先不做大爬虫。

**成功条件：** 范围死、信源死、节奏死、立场可证伪、单一真源 + 固定维护人时、验收=P0 人物 90 天时间线可拉 + 发言可点原文 + 至少 1 次完整 L1/L2。

## 三、核心压缩法：特朗普需求轴（建模假设）

### 用户命题（会话）

库不止沃什五组；还要特朗普、财长等。不研究个体所有细节。核心是：以特朗普为轴，弄清这些人最想要什么、缺什么，由需求反推操作预期；图谱内第一任务默认服务特朗普需求，在此路径下才谈个人需求。

### 评估结论

| 用法 | 判断 |
|------|------|
| 作为研究压缩法 / 委托代理先验 | **合理** |
| 作为全称事实「人人第一任务只服务特朗普」 | **过强** |

更干净表述：

> 在本届行政权主导的政策市场上，多数入选行动者的可见操作高度受 **TrumpDemand(t)** 约束；约束强度随制度位置变化（行政链强、联储部分成立、外部资本常偏离），且必须允许证伪。

### 推荐字段（每人一张，不做百科）

| 字段 | 含义 |
|------|------|
| TrumpDemand 向量 | 当期少数条可观察需求（根节点维护，非特朗普百科） |
| Want | 个人职业/政治资本目标 |
| Lack | 缺权限 / 数据 / 政治掩护 / 市场信任 |
| Alignment | 推进 Demand 的强度：高/中/低/对抗（随事件更新） |
| ExpectedMove | 由上推出的可观察动作 |
| Falsifier | 出现则降低「服务轴」权重的反例 |

### 可证伪检验（摘要）

议程对齐、工具选择、背书型工作组（操作先 vs 年底报告）、市场对独立性定价、利益冲突、数据口径是否服务就业/通胀叙事。  
**能推导的是约束下的操作预期与对齐强度；不能入库的是「内心只忠于特朗普」。**

## 六、最终形态与准确度

### 不应做成

「用大模型扮演特朗普/贝森特/沃什、输出内心独白」——准确度不可控、不可证伪。

### 应做成

**证据型代理人推演系统：** 结构化字段 + 发言证据 + 受约束推理；LLM 最多做抽取/草稿，人确认后入库。

### 准确度控制（可测）

引用完整率、信源等级、ExpectedMove 事前账本命中率、Falsifier 降权、as_of 不穿越、事实/推断/情景分栏、与交易门禁隔离。

### 预期完成度

| 层 | 完成度预期 |
|----|------------|
| L1 台账 | 高（数周） |
| L2 半自动证据 | 中（1–3 月） |
| L3 预期对照有命中率 | 中低（需政策季度样本） |
| L4 情景推演 UI | 中（L3 后） |
| 「人物模拟大模型」 | **不作为目标** |

人物层对交易最多提供情景权重；可证实判断仍靠数据事件 + 微观门禁。

### 主要困难

对抗性表演与改口、可打分样本稀疏、制度黑箱、LLM 幻觉污染、维护衰减、宏微主键未缝合、用「读起来像专家」代替命中率。

---

## 七、已接受决策（本会话）

1. 人物库以 **TrumpDemand 轴 + Want/Lack/Alignment/ExpectedMove/Falsifier** 为压缩核心，不做个体百科。  
2. 「人人第一任务服务特朗普」降级为**默认可证伪先验**，联储/外部资本节点必须允许偏离。  
3. 本阶段只规划 **MVP-0 人工主库**；不抢 NFP 月度循环与微观观察主线。  
4. 终局禁止角色扮演式「模拟人物大模型」作真相引擎。

**是否已被推翻：** 否（2026-07-16）
