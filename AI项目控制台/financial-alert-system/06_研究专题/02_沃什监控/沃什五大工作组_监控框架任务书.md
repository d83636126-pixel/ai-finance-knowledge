## 八、监控系统总体架构

本模块采用“全量归档、条件监控、变化告警、场景重估”的四层结构，不在“仅归档”和“全部告警”之间二选一。

```text
公开信息进入
  → 信源与人物核验
  → Vault 全量归档
  → 判断是否含有效新信息
      ├─ 否：只更新来源记录
      └─ 是：生成监控事件
            → 判断立场或政策方向是否显著变化
                ├─ 否：进入周度摘要
                └─ 是：触发 alert-system 告警
                      → 评估是否需要调整场景概率
```

### 四层产出

| 层级 | 进入条件 | 产出 | 处理时效 |
|---|---|---|---|
| L0 归档 | 与五组、成员或议题相关 | 来源记录 | 7日内 |
| L1 监控 | 出现可验证的新事实或新立场 | 监控卡片 | 48小时内 |
| L2 告警 | 立场显著变化、关键节点确认或原命题被证伪 | 告警事件 | 24小时内 |
| L3 重估 | 信息足以改变资产传导或政策场景 | 场景概率调整记录 | 24小时内 |

## 九、统一可证伪命题模板

五个工作组均使用同一模板，避免只记录“关注什么”而无法在年底判定预测是否命中。

```yaml
group: 数据组
hypothesis_id: WG-DATA-001
created_at: 2026-07-13
baseline:
  direction: 中性偏改革
  policy_prediction: 将建议增加私营或高频替代数据源
  timing_prediction: 正式报告前可能先出现公开讨论
observable_signals:
  - 成员明确提出替代数据源
  - 正式文件讨论实时指标或统计修订
falsifiers:
  - 最终报告明确维持现有数据体系且无实质改革
confidence: 60
evidence_level: B
conflict_of_interest_flag: false
conflict_of_interest_note: ""
review_by: 2026-08-15
status: open
```

### 双重身份（COI）使用规则

- 凡信源人物存在双重身份（如 **Charles I. Jones：Fed 工作组联席 + Anthropic 休假**），监控卡片与命题必须设 `conflict_of_interest_flag: true` 并写明说明。  
- **默认**：相关公开表态按 **B 级**使用（即便场合看似官方）。  
- **恢复 A 级**：仅当独立 Fed 官方渠道（新闻稿/会议文件/国会证词记录）可核验，且已填写 `verification_method`；仍须保留 COI 说明。  
- 「须双重标记」不等于可忽略；未写 COI 的证据不得单独驱动 WARNING/CRITICAL。
### 每组最低命题要求

| 工作组 | 方向命题 | 政策命题 | 时间命题 | 关键证伪条件 |
|---|---|---|---|---|
| 数据组 | 改革/维持现状 | 是否引入替代数据与实时指标 | 报告前是否出现口径讨论 | 最终无数据体系调整 |
| 资产负债表组 | 偏扩张/偏收缩 | QT、准备金与操作框架建议 | 操作是否先于报告变化 | 结论与预测方向相反 |
| 沟通组 | 强/弱前瞻指引 | 声明精简和沟通机制调整 | 操作先行或报告先行 | 报告反对既有操作方向 |
| 生产率就业组 | 供给乐观/审慎 | AI、生产率和就业统计建议 | 是否提前进入公开政策讨论 | 未提出可观测的供给侧调整 |
| 通胀框架组 | 维持/调整框架 | 目标、区间或补偿机制建议 | 是否提前重启框架讨论 | 正式结论完全维持现状 |

> [!warning] 研究纪律
> “背书型工作组”“先斩后奏”等表述属于待验证假设，不得写成已确认事实。每条假设必须绑定证伪条件和复核日期。

## 十、信源等级与证据规范

### 信源等级

| 等级 | 来源 | 使用规则 |
|---|---|---|
| A | 官方报告、正式讲话、国会证词、会议文件、政府网页 | 可直接更新事实字段并触发监控 |
| B | 本人署名论文、本人正式采访、所属机构正式材料 | 可更新立场判断 |
| C | 权威媒体直接引语或完整采访记录 | 可触发监控，关键内容需等待 A/B 级复核 |
| D | 媒体转述、匿名消息、二手报道 | 只作线索，不单独触发场景重估 |
| E | 评论文章、社交媒体推测、无来源摘要 | 仅归档或进入待核验队列 |

### 监控记录必填字段

```yaml
record_id: WG-20260713-001
group:
person:
event_type:
published_at:
captured_at:
source_level:
source_title:
source_url:
verification_method:  # 官网直访 / 官方社交账号 / 二次独立信源 / 国会记录库 等
direct_fact:
direct_quote:
interpretation:
new_information:
related_hypotheses: []
confidence_before:
confidence_after:
scenario_impact:
asset_paths: []
conflict_of_interest_flag: false
conflict_of_interest_note: ""
reviewer:
status: pending_review
```

必须分开记录：

1. **直接事实**：来源明确表达了什么。  
2. **分析解释**：该信息可能意味着什么。  
3. **场景影响**：是否足以改变既有概率或资产路径。

### A/B 级来源真实性核验

判定为 A/B 级时，**必须**填写 `verification_method`。最低要求：

- A 级：官网域名直访、官方会议/证词原文库，或两家独立权威信源交叉确认原文。  
- B 级：本人署名页/机构正式页可核验 URL。  
- AI 生成声明、无法溯源采访、深度伪造嫌疑 → 最高按 E 级归档，不得升 A。

### 信源冲突裁决

| 情形 | 处理 |
|---|---|
| 同级冲突（如两则 A 级结论矛盾） | 状态标 `disputed`；**不**触发场景重估，直至第三方 A/B 出现 |
| 高级覆盖低级 | 允许覆盖，记录中注明「覆盖历史判断」及原 `record_id` |
| 高低级冲突且低级更新 | 低级只作线索；不得单独改评分 |

## 十一、信息采集与触发规则

### 11.1 创建监控事件的条件

满足以下条件才从普通归档升级为监控事件：

```text
人物或工作组命中
+ 议题关键词命中
+ 包含有效新信息
+ 信源达到最低等级
= 创建监控事件
```

### 11.2 有效新信息判定

以下任一情况视为有效新信息：

- 首次确认成员、职责、发布时间或正式议程。
- 成员公开表达新的政策倾向。
- 原有立场出现明显强化、弱化或反转。
- 出现具体政策工具、阈值、时间表或操作建议。
- 正式政策动作与工作组议题产生可验证连接。
- 原有可证伪命题被支持、削弱或否定。

以下情况不单独创建新事件：

- 多家媒体重复报道同一次讲话。
- 旧论文或旧采访被重新转载，但没有新增信息。
- 只出现人物姓名，没有相关政策内容。
- 评论者推测且没有新的原始依据。

### 11.3 去重规则

优先使用以下组合识别同一事件：

```text
人物 + 原始发布日期 + 原始事件名称 + 核心议题
```

同一事件的多家报道合并为一张监控卡片，保留多个来源，但以最高等级信源作为主来源。

## 十二、告警分级与响应动作

| 等级 | 条件 | 动作 |
|---|---|---|
| INFO | 新增相关材料但不改变判断 | 归档，进入周度摘要 |
| WATCH | 出现新立场或新政策线索 | 更新监控卡片和命题置信度 |
| WARNING | 关键时间点确认、立场显著变化或跨组信号共振 | 推送告警，复核资产路径 |
| CRITICAL | 正式结论证伪核心命题，或政策动作足以改变主场景 | 立即重估场景概率并生成决策记录 |

### 告警升级条件

满足任意一项可升级为 `WARNING`：

- A级来源确认正式结论日期或关键制度调整。
- 同一命题获得两个相互独立的 A/B 级来源支持。
- 成员立场变化评分绝对值达到 2。
- 两个以上工作组对同一政策方向形成一致信号。
- 信息影响 BTC、Nasdaq、Gold、USD 或 Treasury 的核心传导路径。

升级为 `CRITICAL` 必须同时满足：

1. A级来源确认；
2. 核心命题被证伪或主场景需要调整；
3. 已写明受影响的传导路径和调整理由。

## 十三、立场变化与场景影响评分

### 13.1 立场变化评分

| 分值 | 含义 |
|---:|---|
| `-2` | 明显转鸽、转向宽松或降低改革力度 |
| `-1` | 轻微向鸽派或保守方向移动 |
| `0` | 无有效变化 |
| `+1` | 轻微转鹰、收紧或增强改革力度 |
| `+2` | 明显转鹰、转向收紧或发生制度性变化 |

评分必须附带 A–C 级证据；D/E 级来源不得单独改变评分。

### 13.2 场景影响评分

| 分值 | 含义 | 处理方式 |
|---:|---|---|
| 0 | 不影响现有场景 | 仅归档 |
| 1 | 提供轻微信号 | 更新周报 |
| 2 | 明显支持或削弱次要场景 | 更新命题置信度 |
| 3 | 影响主场景或核心资产路径 | 触发重新打分 |

场景概率调整必须记录调整前、调整后、变化幅度和直接证据，不允许只写方向判断。

### 13.3 场景概率回撤（只追加、不覆盖）

L3 调整后若发现误判或触发源被证伪：

1. **禁止**静默改写旧记录。  
2. 新建一条调整记录，填写：

```yaml
superseded_by: WG-SCENARIO-YYYYMMDD-00N
superseded_reason: 触发源被证伪 / 误判 / 信源降级
```

3. 旧记录保留，状态改为 `superseded`；新记录写明回撤后的概率。

## 十四、年底预测偏差评分

原有 1–5 分偏差度改为结构化评分，减少事后主观判断。

### 14.1 单组内权重

| 评分项 | 权重 | 判定内容 |
|---|---:|---|
| 方向命中 | 35% | 鹰/鸽、扩张/收缩、改革/维持方向是否一致 |
| 政策命中 | 30% | 是否命中具体制度或政策建议 |
| 关键论据命中 | 20% | 对结论形成原因的判断是否准确 |
| 时间窗口命中 | 15% | 信号和正式结论出现时间是否符合预测 |

### 14.2 组间权重（整体方法论命中率）

| 优先级 | 工作组 | 权重 |
|---|---|---:|
| P0 | 数据组 | 30% |
| P1 | 资产负债表组 | 25% |
| P2 | 沟通组 | 20% |
| P3 | 生产率就业组 | 15% |
| P4 | 通胀框架组 | 10% |

```text
单组命中率 = 四项加权得分
整体方法论命中率 = Σ (单组命中率 × 组间权重)   # 仅计入已发布且非 pending 的组，权重再归一化
结论偏差度 = 100 - 命中率
校准误差 = 初始置信度 - 实际命中结果
```

### 14.3 复盘触发规则

1. **硬性复盘日：每年 1 月 15 日**（对上一自然年工作组结论窗口）。  
2. 届时仍未发布的组：标记 `pending`，**不计入**整体命中率分母（权重对已发布组重新归一化）。  
3. 硬性日后 **3 个月内**补发：只做**单组附录复盘**，不追溯改写已发布的整体主报告。  
4. 可选软触发：末组发布后 14 日内可提前出主报告，但仍保留 1/15 作为未发布组的截止基准。

### 14.4 偏差等级

| 偏差度 | 等级 | 含义 |
|---:|---|---|
| 0–20 | 1 | 高度吻合 |
| 21–40 | 2 | 大体吻合，局部偏差 |
| 41–60 | 3 | 部分命中，关键环节不确定 |
| 61–80 | 4 | 大部分判断失准 |
| 81–100 | 5 | 核心方向完全背离 |

年底复盘必须分别评价“预测是否错误”和“当初置信度是否过高”，二者不得混为一谈。

### 14.5 命题过期（stale）处置

| 阶段 | 规则 |
|---|---|
| `review_by` 到期仍 `open` | 自动转 `status: stale` |
| stale 后 7 日内 | 必须人工重设 `review_by` 或明确关闭（hit / miss / withdrawn） |
| stale 超过 30 日未处理 | 年底偏差评分计为 **未验证**：不进命中率分子与分母 |
