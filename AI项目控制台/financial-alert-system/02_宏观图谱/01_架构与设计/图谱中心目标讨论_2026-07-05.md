# 图谱中心目标讨论

## 背景

当前图谱已经有多个目标节点：

- `btc_price`：BTC 价格
- `eth_price`：ETH 价格
- `spx`：标普500
- `gold_price`：黄金
- `altcoin_season`：山寨币综合指数

同时，以下关键节点目前是中间节点：

- `risk_appetite`：风险偏好
- `usd_index`：美元指数 DXY
- `treasury_yield`：10年期美债收益率
- `nasdaq`：纳斯达克指数

## 核心问题

图谱不能只问“最后哪个资产涨跌”，还要先确定中心目标是什么。

中心目标决定：

- 哪些节点应该优先补齐。
- 哪些边最重要。
- 权重如何校准。
- 验证节点应该验证什么。
- 预警输出应该长什么样。

## 三种可选中心目标

### 方案 A：以 BTC 为中心

图谱主要回答：

```text
全球宏观、美元流动性、风险偏好、链上、衍生品如何影响 BTC？
```

优点：

- 目标清晰。
- 与 crypto / onchain / derivatives 节点天然匹配。
- 适合做加密资产预警。

缺点：

- 全球宏观节点容易变成 BTC 的附属解释。
- 对 SPX、黄金、美元、美债的解释能力会弱一些。

### 方案 B：以风险偏好为中心

图谱主要回答：

```text
当前全球市场处于 risk-on 还是 risk-off？
```

然后再传导到：

- BTC
- ETH
- Nasdaq
- SPX
- Gold
- USD
- Treasury

优点：

- 更适合全球宏观预警。
- 能同时解释多个资产。
- 适合把美国、中国、欧洲、日本、能源、地缘节点纳入统一框架。

缺点：

- 需要把“风险偏好”定义清楚。
- 输出比单资产预测复杂。

### 方案 C：以资产篮子为中心

图谱主要回答：

```text
同一个宏观冲击下，不同资产分别如何反应？
```

目标资产包括：

- BTC
- ETH
- Nasdaq
- SPX
- Gold
- USD
- Treasury
- Oil

优点：

- 更像完整金融预警系统。
- 能输出跨资产影响矩阵。

缺点：

- 第一版复杂度较高。
- 每个资产都需要校准路径和验证指标。

## 已确认方向

推荐采用：

```text
中心目标 = 全球风险状态
输出目标 = 资产篮子
重点资产 = BTC + Nasdaq + Gold + USD + Treasury
```

也就是：

```text
全球事件 / 数据 / 政策
  → 美元流动性、利率、通胀、增长、地缘、风险偏好
  → BTC / Nasdaq / Gold / USD / Treasury
```

这样图谱既不会只服务 BTC，也不会变成没有焦点的百科式图谱。

## 设计建议

### 1. 把 `risk_appetite` 升级为核心状态节点

它不一定是最终 target，但应作为全图核心 hub。

### 2. 把资产目标分层

核心输出：

- BTC
- Nasdaq
- Gold
- USD
- Treasury

次级输出：

- ETH
- SPX
- Altcoin Season
- Oil
- Emerging Markets

### 3. 每个目标资产都要有验证指标

示例：

- BTC：ETF flow、funding、open interest、onchain liquidity
- Nasdaq：real yield、earnings expectation、VIX
- Gold：real yield、DXY、geopolitical risk
- USD：rate differential、funding stress
- Treasury：inflation expectation、Fed path、term premium

- 第一版以“全球风险状态”为中心。
- BTC 仍是重要输出资产，但不是唯一中心。
- Nasdaq / Gold / USD / Treasury 作为第一版核心资产篮子。

## 项目决策

从 2026-07-05 起，`financial-alert-system` 的图谱设计不再以单一资产作为中心，而是采用：

```text
中心目标 = 全球风险状态
输出目标 = 资产篮子
重点资产 = BTC + Nasdaq + Gold + USD + Treasury
```

后续节点补齐、边设计、权重校准、验证节点和预警输出，都应围绕这个中心目标展开。

## 2026-07-05 补充决策：Nasdaq 升级为 target

用户确认将 `nasdaq` 从 `intermediate` 升级为 `target`。

原因：

- Nasdaq 是第一版核心输出资产篮子的一部分。
- 系统需要直接输出 Nasdaq 的风险方向，而不是只把 Nasdaq 当成 BTC / SPX 的中间传导变量。
- 这与“全球风险状态 -> 资产篮子”的中心目标一致。

执行结果：

```text
nasdaq.type: intermediate -> target
nasdaq.subtype: equity_index
```

## 2026-07-05 补充决策：USD / Treasury 目标表达

用户确认处理 USD / Treasury 的 target 表达。

执行方向：

```text
usd_index -> 升级为 target
treasury_yield -> 保持 intermediate
treasury_bond_return -> 新增为 target
```

原因：

- USD 可以直接由 DXY 表达。
- Treasury 不宜直接用收益率作为 target，因为收益率与债券价格/回报方向相反。
