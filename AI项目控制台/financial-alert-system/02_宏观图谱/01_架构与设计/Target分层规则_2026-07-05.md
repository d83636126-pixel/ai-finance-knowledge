# Target 分层规则 2026-07-05

## 背景

图谱当前已有多个 target。为了让 UI、预警输出、回测和解释报告有主次，需要区分：

```text
核心 target
扩展 target
```

## 规则

### 核心 target

核心 target 是第一版预警系统必须优先输出的资产篮子：

```text
BTC
Nasdaq
Gold
USD
Treasury
```

对应节点：

```text
btc_price
nasdaq
gold_price
usd_index
treasury_bond_return
```

### 扩展 target

扩展 target 是第二层输出，用于补充观察、交叉验证或后续版本展开：

```text
ETH
SPX
Altcoin Season
```

对应节点：

```text
eth_price
spx
altcoin_season
```

## 已写入字段

已在 `propagation_graph.json` 的 target 节点中写入：

```text
target_tier
target_priority
```

当前优先级：

```text
btc_price: core, 1
nasdaq: core, 2
gold_price: core, 3
usd_index: core, 4
treasury_bond_return: core, 5
eth_price: extended, 101
spx: extended, 102
altcoin_season: extended, 103
```

## 校验结果

- JSON 校验：通过
- 图谱健康评分：88
- 图谱状态：watch
- 孤立节点：0
- target 节点数：8

## 后续用途

后续 UI 可以：

- 默认显示核心 target。
- 将扩展 target 放在折叠区或次级面板。
- 预警摘要优先输出核心 target。
- 回测可以先跑核心 target，再扩展到第二层。
