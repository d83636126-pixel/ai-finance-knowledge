# 核心 Target 验证仪表

## 背景

上一轮已经让路径报告和 Case 报告的 target 卡片显示验证指标和失效条件。

本轮继续把它前置成一个更清晰的“核心 target 验证仪表”，让用户在报告顶部先看到：

```text
BTC / Nasdaq / Gold / USD / Treasury
当前方向
验证点数量
关键验证指标
最重要失效条件
```

## 本次实现

已更新：

```text
propagation.html
```

新增函数：

```text
renderTargetValidationDashboard(targets)
```

接入位置：

- FOMC Case 报告摘要下方。
- 普通路径报告摘要下方。

## 展示逻辑

仪表只展示核心 target：

```text
target_tier = core
```

按 `target_priority` 排序。

每个资产卡片显示：

- 当前预期方向。
- 验证点数量。
- 路径数量。
- 最佳权重。
- 前 3 个验证指标。
- 1 条失效条件。

## 后续建议

下一步可以做浏览器人工检查：

- FOMC Case 报告顶部是否能快速看清核心资产。
- 验证指标是否过多或过少。
- 失效条件是否需要改成更接近交易/研究语言。

之后再考虑把仪表做成交互式：

- 点击 BTC 只看 BTC 验证点。
- 点击 Nasdaq 只看 Nasdaq 验证点。
- 点击 Gold / USD / Treasury 看对应链条。
