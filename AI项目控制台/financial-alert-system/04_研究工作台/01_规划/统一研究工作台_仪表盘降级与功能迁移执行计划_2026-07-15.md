# 统一研究工作台：仪表盘降级与功能迁移执行计划

> [!success] 已确认方向
> 独立仪表盘停止扩展，统一研究工作台成为唯一主入口。有效功能迁入，工程信息折叠，无真实来源或重复能力归档。通过真实数据、用户价值和回退验收前，不切换根入口、不删除旧页面。

## 一、目标与边界

工作台打开后应在 10 秒内回答：

1. 现在发生了什么；
2. 最新重大新闻及来源是什么；
3. 市场和微观结构如何反应；
4. 它怎样改变宏观推导和传导路径；
5. 当前结论是否可信、下一步验证什么。

产品链路固定为：`实时事实 → 宏观推导 → 微观验证 → 传导路径 → 证据复核`。

- 保留研究建议、风险提示和验证任务。
- 不加入下单、账户授权、目标仓位、自动执行或自动交易。
- 不改变 kill switch，不修改正式校准簿，不绕过候选门禁。
- 已完成的微观总览、微观工作台和传播图作为组件复用，不重复重做。

## 二、统一工作台结构

| 模式 | 作用 | 默认状态 |
|---|---|---|
| 实时研究总览 | 关键变化、重大新闻、市场反应、当前推导、待验证项 | 首屏 |
| 新闻与事件 | 可追溯事件流、来源、发布时间、采集时间、关联资产 | 主导航 |
| 宏观推导 | 事实、假设、证据、反证、情景、失效条件 | 按需 |
| 微观验证 | 价格、成交、波动率、期限结构、基差、持仓验证 | 按需 |
| 传导路径 | 事件到状态与资产的聚焦路径、替代路径 | 按需 |
| 证据检查 | 来源、时间、角色、冲突、缺口、验证状态 | 按需 |
| 历史回放 | 当时快照、推导、事后结果和复盘 | 按需 |
| 系统健康 | 数据源、延迟、采集器、P3、错误和降级 | 默认折叠 |

迁移期先建立 `/workbench.html`，用 `?mode=overview/news/macro/micro/transmission/evidence/history/health` 切换模式；`/index.html` 在验收前保持原状。

## 三、旧仪表盘功能迁移表

| 旧功能 | 处理 | 新位置 / 条件 |
|---|---|---|
| 当前关注、告警 | 迁入 | 实时总览与新闻；改成可追溯研究关注项 |
| 宏观时间线 | 迁入并简化 | 新闻与历史回放；实验性视图归档 |
| 事件分析 | 迁入 | 新闻详情与宏观推导；事实和模型判断分开 |
| 活跃、紧急、简报计数 | 折叠 | 总览摘要条；必须可点回明细 |
| 微观摘要 | 迁入 | 微观验证；复用现有组件 |
| 评分全景 | 条件保留 | 有定义、输入、时间和版本才保留，否则归档 |
| AI 信号 | 改名后条件迁入 | 改为研究发现或待验证命题，不输出交易信号 |
| RSS、预热、预算、系统状态 | 折叠 | 系统健康；异常才提升提示 |
| 绩效、准确率、混淆矩阵 | 迁移 | 历史回放 / 方法论，不占实时首屏 |
| 情绪、相关性热图 | 条件保留 | 有真实来源、窗口和更新时间才迁入微观验证 |
| 设置与 API Key | 迁移 | 系统健康 → 配置；绝不回显密钥 |
| 重复弹窗、空占位、内部字段 | 归档 | 根入口切换后移除引用，不立即删源文件 |

每项旧功能必须登记为 `迁入 / 折叠 / 条件保留 / 归档`；去向未明，不得切根入口。

## 四、实时数据和新闻硬门槛

每条正式信息至少包含：

```text
source
source_url / series_id / instrument_id
event_time 或 observation_time
published_at（适用时）
ingested_at
role = live | restored | placeholder | fixture | derived
freshness_status = fresh | delayed | stale | unknown
delay_seconds
```

- restored、placeholder、fixture 不得显示成 live。
- 新闻缺来源、发布时间或采集时间，不进入正式事件流。
- 接口失败显示不可用或延迟，不用旧值静默顶替实时值。
- 派生结论必须回溯输入、算法版本和计算时间。
- 主要市场价格目标 5–15 秒；微观数据 30–60 秒；官方新闻 1–2 分钟；计划内宏观发布 30–60 秒。超时必须显式降级。

## 五、Cursor 分阶段执行

### UW0：冻结现场与迁移台账

- 记录分支、HEAD、`git status --short` 和未提交文件归属。
- 截图并登记 `/index.html`、`/propagation.html`、`/micro-layer/index.html`、`/micro-layer/workbench.html`。
- 逐项完成旧功能处置表，标出可复用组件、接口和回退路径。
- 旧仪表盘只允许阻断性修复，不再新增能力。

交付：`统一研究工作台_UW0现场基线与迁移台账_2026-07-15.md`。

门禁：旧功能均有去向；现场改动均有归属；回退路径可执行。**Cursor 首次只执行 UW0，报告写回前不得进入 UW1。**

### UW1：实时事实与新闻真相层

- 盘点市场、宏观、新闻、微观数据源的免费 / 收费、频率、许可和失败策略。
- 补齐来源、时间、角色、新鲜度字段和统一判定。
- 建立新闻去重、事件聚合、来源优先级和降级规则。
- 测试缺来源、缺时间、过期、恢复数据和接口失败。

门禁：任取页面信息均能回答来源、发生时间、采集时间和实时状态；缓存或占位不参与正式判断。

### UW2：统一外壳

- 建立 `/workbench.html`、八个模式、统一事件选择器、时间锚点和来源状态条。
- 通过适配层复用微观总览、微观工作台和传播图，不复制业务逻辑。
- 保留旧页面兼容入口和返回路径。

门禁：模式可直链和刷新恢复；上下文不丢失；旧入口仍可用；根入口未切换。

### UW3：实时总览与新闻事件流

- 首屏展示关键变化、重大新闻、真实市场反应、宏观影响、可信度和下一验证点。
- 研究发现与正式事实分开；空数据、延迟和来源冲突明确显示。
- 每条关键内容可进入证据详情，并连接宏观、微观和传导模式。

门禁：正常本地环境首个有意义内容 2 秒内出现；1366×768 首屏回答五问；至少一个真实事件端到端通过。

### UW4：宏观—微观—传导—证据闭环

- 宏观显示事实、假设、证据、反证、情景和失效条件。
- 微观输出确认、部分确认、否证或数据不足，并解释依据。
- 传导只聚焦当前事件相关路径；证据统一显示来源、时间、角色、冲突和缺口。
- 支持从一条事件连续进入宏观、微观、传导并返回原上下文。

门禁：真实事件完成闭环；任一结论可回溯；数据不足不输出确定判断；C10/C11 语义不回退。

### UW5：系统健康折叠与遗留收口

- 数据源、采集器、RSS、P3、预算、缓存、错误迁入系统健康。
- 正常折叠，异常显示全局可点击提示。
- 评分、情绪、热图、绩效逐项“验真后迁入，否则归档”。
- API Key 移出研究界面并验证不回显。

门禁：工程状态可发现但不淹没首屏；每项旧功能有最终去向和证据。

### UW6：逐项验收

逐项验证功能、数据、研究语义、10 秒用户任务、1366×768 / 1440×900 / 移动窄屏、性能、密钥和交易边界。

验收状态只允许 `通过 / 有条件通过 / 阻断 / 未验收`；不得用一个总分掩盖阻断。所有 P0/P1 通过、占位不参与正式判断、根入口回退演练成功，才可进入 UW7。

### UW7：根入口切换与旧仪表盘归档

- 旧根仪表盘保存为 `archive/index_dashboard_legacy_2026-07-15.html` 或等价只读归档。
- `/index.html` 切换到统一工作台；保留传播图和微观层兼容深链。
- 更新启动说明、导航、测试、Obsidian 和变更记录。
- 验证静态引用、API、冒烟、采集器、P3 和一键回退。

### UW8：24 / 72 小时观察

- 24 小时观察采集连续性、页面错误、新鲜度、新闻去重、链接和用户任务。
- 72 小时观察数据源稳定性、降级频率、事件闭环质量和旧入口需求。
- 只有用户再次明确确认，才物理删除旧页面或模块。

执行顺序：`UW0 → UW1 → UW2 → UW3 → UW4/UW5 → UW6 → UW7 → UW8`。

## 六、执行纪律与硬停止

每阶段开始前读取本计划、上一阶段报告和当前 Git 状态，列出精确改动文件；结束后写回改动、测试、浏览器证据、未通过项、回退和下一门禁。

禁止 `git reset --hard`、`git checkout --`、`git clean`、整文件覆盖他人改动、`git add .`、未经确认自动推送，以及把 restored / placeholder / fixture 冒充实时值。

出现以下任一项立即停止：占位进入正式结论；旧值冒充实时；新闻缺来源或时间；结论不可回溯；关键旧功能去向不明却准备切根入口；P3、采集器或微观链路回归；正式校准簿被改；候选门禁被绕过；出现订单、账户授权、目标仓位或自动执行能力。
