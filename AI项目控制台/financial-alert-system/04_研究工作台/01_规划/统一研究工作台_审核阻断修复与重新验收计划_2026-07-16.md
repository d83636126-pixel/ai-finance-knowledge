## 2. 修复目标与不可变边界

### 2.1 修复目标

1. 关闭凭据、隐藏目录和未授权 API 暴露；
2. 新浏览器能主动获得真实新闻和市场数据，不依赖历史 localStorage；
3. stale、restored、fixture、manual、derived 永远不冒充实时事实；
4. 同一真实事件贯通新闻、宏观、微观、传导与证据；
5. 宏观证据缺失时 fail-closed；
6. 重建不可变、可复核的 UW6-v2 验收；
7. 建立真实可执行的回退路径；
8. 从稳定 Git 提交重新完成 24/72 小时观察；
9. 让源码状态、工件、Obsidian 和 Git 口径一致。

### 2.2 不可变边界

- 项目继续聚焦宏观推导与微观分析，不向自动交易系统延伸；
- 不加入券商、交易所账户授权、订单提交、目标仓位和自动执行；
- `advisory_only=true`、`executable=false`、kill switch=true；
- 正式校准簿默认只读，任何测试必须记录前后哈希；
- 不使用 `git reset --hard`、`git checkout --`、`git clean`；
- 不使用 `git add .`，不自动提交或推送；
- 不输出、记录或回显任何真实密钥。

## SEC0：静态服务与凭据安全封口

**涉及：** `local_server.js`、所有 HTML、本地配置加载、鉴权与安全 smoke。

最小改动：

- 静态文件改为显式公开清单，只允许必要 HTML、CSS、JS、图片和经批准的数据文件；
- 永久拒绝 dotfile、`.env`、`.git`、`node_modules`、`static/local_config.js`、日志与源码运维目录；
- 用 `path.relative()` 判断路径边界，禁止字符串前缀校验与编码穿越；
- 浏览器移除所有本地密钥加载；FRED、翻译等只经服务端 BFF；

- `/api/proxy`、`/api/translate` 增加鉴权、超时、并发、响应大小和速率限制；
- 如果服务曾通过非回环、隧道或云端开放，单独列出需轮换的密钥，由用户执行外部轮换。

测试：

- `/.env`、`/.git/config`、`/static/local_config.js` 与编码目录穿越均返回 403/404；

- 非回环无授权启动或 API 访问被拒；
- 下载资产扫描不出现密钥特征；测试输出不得包含密钥正文。

硬停止：任一敏感路径仍返回 200、任一未授权 API 可用、任一页面继续加载本地密钥配置。

## RT1：真实新闻与市场刷新控制器

**涉及：** 统一入口、EventIngestor、MarketSnapshotUpdater、VerifierDataFetchers，建议新增单一 realtime controller。

最小改动：

- 根入口启动后实际执行一次 RSS 扫描与市场刷新；
- 由一个控制器统一管理新闻、价格、市场状态与宏观数据，不在多个组件重复轮询；
- 建议频率：价格 15–30 秒、市场状态约 60 秒、新闻 60–120 秒、日频宏观 5–15 分钟；
- 增加请求超时、指数退避、429 冷却、单飞锁和页面隐藏降频；
- 监听 `inbox-changed`、storage、online 和 visibility；
- fresh browser 不依赖 demo、fixture 或已有 localStorage；
- 失败时只能进入 restored / stale / unavailable，不得把旧值写成 live。

测试：

- 空浏览器启动后实际出现 RSS 与市场请求；
- 至少两个独立新闻源和一个市场源完成真实端到端；
- 记录 source、source_url、published_at、ingested_at、as_of 和 delay；
- 失败、超时、429、恢复与页面隐藏均有确定行为。

硬停止：新浏览器仍无请求、全部来源 stale、出现请求风暴、失败后旧值变 live、demo 进入正式流。

## TRUTH1：实时事实与新闻资格契约

**涉及：** `realtime_truth.js`、`event_ingestor.js`、`unified_workbench.js`、市场状态映射和真实性测试。

最小改动：

## 7. 全局硬停止条件

出现任一项立即停止，不得推进下一阶段或升级状态：

- 敏感文件可通过 HTTP 下载；
- 未授权 API 可用；
- fresh browser 无 RSS / 市场请求；
- 正式新闻为 0 却宣称实时新闻通过；
- stale / restored / fixture / manual / derived 显示为实时事实；
- 真实事件闭环为 0；
- 下游忽略 event_id / as_of 后仍输出结论；
- 宏观无桥上下文得到 confirmed；
- 人工清单未填写或回退演练失败；
- 无稳定 commit / tag；
- 24 / 72 小时时长未实际达到；
- actionable、kill switch 关闭、订单尝试或正式校准簿变化。
