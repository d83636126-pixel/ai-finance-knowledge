# financial-alert-system：最终用途与路线图

## 最终用途（已确认）

> **宏观传导研究 + 预警能力补强的混合系统。**
>
> - 图谱用于补全 Alert 侧「识别范围不足、缺宏观链路」的问题。
> - 核心场景：**事前分析** + **事后对比分析**。
> - **推送功能延后**，等其他能力闭环后再做。

一句话：

**先用图谱把宏观链路讲清楚、验清楚；扫描与推送是下游，不是第一优先级。**

## 产品分层

| 优先级 | 内容 | 状态 |
|--------|------|------|
| **P1** | 传播图、事前 Case、事后 evidence 对比、验证数据源 | 进行中 |
| **P2** | Alert ↔ 图谱桥接、扫描范围按 macro 链路扩展 | 图谱稳定后 |
| **P3** | 微信推送、Cron 7×24 | **明确延后** |

## 第一版最小闭环（验收）

一次完整的 **事前 → 事后** 闭环：

1. **事前**：选 FOMC 鹰派（或日历事件）→ 五资产路径 + 验证清单 + 失效条件
2. **数据**：至少 3 个 verifier 有真实数据（TIPS、DXY、MOVE）
3. **事后**：填入/自动拉 snapshot → evidence 状态更新
4. **对比**：事前预期方向 vs 事后实际（第一版可半手动）

## P1 任务清单

### A. 计划内事件（日历驱动）

- [x] 统一 `MAJOR_EVENTS` 数据格式（`MajorEvents` 归一化；timeline 优先读 CONFIG）
- [x] 传播图「日历 overlay」与 `ScenarioReasoner` 事件库打通（CalendarReasonerBridge）
- [x] 事前：从日历选事件 → 生成 pre-report（面板内结果 + 可跳转推演）
- [x] 事后：填入 actual outcome → 生成 post-report + 对比
- [ ] 扩展 Case 库：非农、CPI、地缘（不限 FOMC）

### B. 突发事件（见 [[AI项目控制台/financial-alert-system/突发事件处理_2026-07-10]]）

- [ ] 事件摄入层：**中英文 RSS** + 关键词 + inbox（**手动触发分析**）
- [ ] LLM 归类助手：`EventClassifier` 建议 graph node，**人确认后再跑路径**
- [ ] 症状触发层：市场异动 → 提示「可能对应哪类 graph event」
- [ ] 快速录入：手动一键创建突发 event instance → 跑路径报告
- [ ] 事后对比：与计划内事件同一套 pre/post 流程

### C. 数据佐证层

- [ ] 优先接 FRED（TIPS）、DXY、MOVE（第一版 3 源）
- [ ] market snapshot 手动填写入口（或 JSON 模板）
- [ ] evidence 状态自动计算（confirmed / partial / invalidated）

### D. 明确不做（P1）

- [ ] ServerChan / 微信推送
- [ ] Cron 无人值守扫描
- [ ] 后端化 Scanners
