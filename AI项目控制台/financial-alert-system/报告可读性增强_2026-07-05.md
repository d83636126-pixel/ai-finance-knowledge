# 报告可读性增强

## 背景

图谱已经完成：

- 核心 target：BTC / Nasdaq / Gold / USD / Treasury
- 二级传导：美元、美债、实际利率、黄金、BTC
- 第三层验证指标：TIPS、MOVE、DXY、ETF flow、纳指宽度等

本轮目标是让路径报告不只是列出路径，而是让用户一眼看到：

```text
这个目标资产为什么受影响？
用什么指标验证？
什么情况说明这条路径失效？
```

## 本次实现

已更新：

- `static/propagation_engine.js`
- `propagation.html`

实现内容：

- 路径报告会自动从图谱 `verifies` 边收集验证指标。
- 每条路径会携带自己的 `validation_points`。
- 每个 target 资产卡片会携带自己的 `validation_points`。
- target 验证点会自动去重。
- 起点事件验证点不再混入 target 卡片，避免把“事件是否成立”误当成“资产路径是否确认”。
- 页面资产卡片显示最多 3 个验证指标。
- 页面资产卡片显示一条失效条件。
