# financial-alert-system：需求拆分

## 项目目标

建立一个 **宏观传导研究 + 预警补强** 的混合系统：用传播图补全 Alert 在宏观链路上的不足，支持 **事前分析** 与 **事后对比**；推送功能延后至 P3。

详见 [[AI项目控制台/financial-alert-system/最终用途与路线图_2026-07-10]]。

## 第一版大需求

- 让系统能稳定本地运行。
- 让核心页面入口清晰。
- 让传播图数据结构清晰、可检查、可维护。
- 让 AI 能基于项目文档继续分析和开发。

## 功能拆分

### 1. 项目基础

- [ ] 确认项目运行命令。
- [ ] 确认主要页面：`index.html`、`propagation.html`、`architecture.html`。

### 2. 传播图数据

- [ ] 梳理 `propagation_graph.json` 的节点类型。
- [ ] 梳理 `propagation_graph.json` 的边类型。
- [ ] 检查是否符合 `graph_schema.md`。
- [ ] 建立图谱健康检查流程。
- [ ] 扩展非美国全球关键节点，详见 [[AI项目控制台/financial-alert-system/图谱节点扩展_全球非美国节点]]。
- [ ] 明确图谱中心目标，详见 [[AI项目控制台/financial-alert-system/图谱中心目标讨论]]。

- [ ] 基于缺口对照确认第一批新增节点，详见 [[AI项目控制台/financial-alert-system/图谱节点缺口对照_2026-07-05]]。

### 3. 页面体验

- [ ] 明确首页用途。
- [ ] 明确传播图页面用途。
- [ ] 明确架构页面用途。
- [ ] 检查本地静态资源是否完整。

### 4. 事前 / 事后分析（P1 核心）

- [ ] 计划内事件：日历 → pre-report → post-report → compare（复用 ScenarioReasoner）
- [ ] 突发事件：event inbox → 图谱映射 → 即时路径报告 → 事后 evidence 对比
- [ ] 详见 [[AI项目控制台/financial-alert-system/突发事件处理_2026-07-10]]

### 5. 事件抓取（P1，不推送）

- [ ] RSS + BREAKING_KEYWORDS 规则匹配（config 已有，需接入扫描）
- [ ] event_taxonomy.json：新闻关键词 → graph event 映射
- [ ] 市场症状触发：多信号聚合 → inbox（symptom_trigger）
- [ ] 人工快速录入突发

### 6. 预警逻辑（P2/P3，延后）

- [ ] 确定预警输入：宏观、链上、衍生品、政策、流动性。
- [ ] Alert ↔ 图谱桥接：信号映射到 graph event。
- [ ] 推送（ServerChan）：**P3，其他功能完善后再做**。

### 5. Obsidian 联动

- [ ] 将项目需求和进度记录在 Obsidian。
- [ ] 将关键图谱结论链接到金融知识库笔记。
- [ ] 每次开发结束更新会话交接。

## 暂不做

- 暂不接真实交易接口。
- 暂不做自动下单。
- 暂不承诺预测准确率。
- 暂不大改技术栈。
