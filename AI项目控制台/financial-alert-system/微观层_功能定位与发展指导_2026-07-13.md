# 微观层：功能定位与发展指导（2026-07-13）

## 一句话定位

> 微观层不是价格预测器，也不是自动下单机器人；它是宏观传播结论进入仓位决策前的**市场验证器、微观风险雷达、组合约束入口和强制弃权门禁**。

它真正解决的问题是：

> 把交易所、衍生品、资金流和链上微观数据，转换成一套可验证、可回溯、可弃权、受风险约束的仓位决策依据。

## 在系统中的位置

```mermaid
flowchart LR
    A["宏观层<br/>为什么市场可能变化"] --> B["传播图<br/>影响沿什么路径传递"]
    B --> C["微观层<br/>市场内部是否真实响应"]
    C --> D["组合层<br/>账户是否承受得起"]
    D --> E["执行门禁<br/>允许建议或必须弃权"]
```

宏观层回答“理论上为什么变化”，传播图回答“影响如何传递”，微观层回答“市场内部是否已经出现可验证的响应”。因此，微观层不是重复宏观分析，而是宏观推导与实际风险暴露之间的最后一公里。

## 实际目的

### 1. 验证宏观推导是否落到市场

当宏观层判断美元、利率、流动性或风险偏好可能影响 BTC、ETH、SOL 时，微观层继续检查：

- 价格趋势是否形成；
- OI 是新增建仓还是去杠杆；
- funding 是否过度拥挤；
- 多家交易所是否互相印证；
- 期权、资金流和链上结构是否支持；
- 数据来源是否足够可信。

最终只允许三类结论：

1. 宏观与微观一致，假设得到增强；
2. 宏观与微观冲突，降低置信度；
3. 数据或证据不足，保持观察并弃权。

### 2. 识别价格背后的市场结构

同样的价格上涨，可能来自健康建仓、杠杆拥挤、空头回补或低流动性推动。微观层必须区分这些结构，而不是把“上涨”直接翻译成“看多”。

| 表面现象 | 微观结构 | 决策含义 |
|---|---|---|
| 价格上涨、OI 上升、funding 温和 | 新增多头参与 | 趋势可能较健康 |
| 价格上涨、OI 暴增、funding 极端 | 多头杠杆拥挤 | 回撤和清算风险上升 |
| 价格上涨、OI 下降 | 空头回补 | 不一定存在持续买盘 |
| 价格上涨、成交弱、跨所不一致 | 证据不足 | 不宜追随 |

### 3. 防止重复信号制造虚假确定性

funding、OI、清算、长短仓比可能都在表达“杠杆拥挤”。系统必须先形成状态节点，再按因子去重；同一因子只能贡献一次，不能因为同类指标数量多就人为提高置信度。

### 4. 把研究判断转换为受约束的仓位建议

微观层输出仓位建议前，必须依次回答：

1. 数据是否完整、新鲜且单位一致；
2. 来源是否可信，多交易所是否一致；
3. 状态是否经过历史校准；
4. 当前市场状态是否匹配历史样本；
5. 账户现有暴露和相关性是否允许；
6. 流动性、滑点、单资产及总杠杆限制是否通过。

任一硬条件不满足，仓位变化必须为空并输出弃权原因。微观层最重要的能力之一，是解释“为什么现在不能动”。

### 5. 为未来执行建立不可绕过的门禁

当前及可预见阶段保持 `advisory-only`。即使未来接入模拟盘或券商，也必须保留：

- 默认不执行；
- 人工授权与有效期；
- 校准、组合和流动性门禁；
- 杀开关；
- 禁止 LLM 或 `force` 字段越权；
- 完整审计记录。

门禁通过只代表“允许进入下一道检查”，不代表自动提交订单。

## 核心使用场景

### 事件窗口监控

用于 CPI、非农、央行决议、ETF 资金流、监管和地缘事件发生后，观察宏观冲击是否真实进入加密市场内部。

### 微观风险预警

识别 funding 极端、OI 快速扩张或坍缩、跨所分裂、流动性恶化、期权波动率冲击和潜在去杠杆风险。

### 传播路径验证

判断宏观传播路径处于“已确认、部分确认、冲突、尚未发生或证据不足”中的哪一状态。

### 历史研究

积累“因子 × 市场状态 × 标的 × 持有期”的样本，验证哪些结构具有可重复的统计价值，并记录失效条件。

### 组合风险控制

避免 BTC、ETH、SOL 等高相关资产同时扩大暴露，控制相关性簇、目标波动率、单资产风险和总杠杆。

## 标准输出

成熟的微观层应稳定输出五类结果：

1. **市场状态**：杠杆拥挤、空头挤压、OI 扩张、去杠杆、波动率冲击等；
2. **数据可信度**：来源等级、跨所一致性、字段缺口、代理数据和熔断状态；
3. **假设验证**：宏观结论得到确认、发生冲突、尚未兑现或无法判断；
4. **风险建议**：保持、降低、允许小幅增加、等待确认或强制弃权；
5. **审计链路**：原始数据 → 归一化 → 状态 → 因子 → 校准 → 组合约束 → 结论。

## 明确的非目标

微观层不得被发展为：

- 精确预测下一根 K 线的机器；
- 单指标买卖信号器；
- 高频交易系统；
- 由 LLM 直接控制的自动下单机器人；
- 用几个指标替代完整风险管理的捷径；
- 把代理数据包装成真实清算、链上持仓或资金流事实的展示层。

## 后续发展的指导原则

### 原则一：先可信，再丰富

增加新指标前，优先保证现有 funding、OI、价格和来源健康数据在单位、时间、新鲜度和跨所口径上可靠。指标数量不是完成度。

### 原则二：完备度与可信度分开

`data_quality` 回答字段是否齐全，`source_trust` 回答来源是否可信。两者是正交门禁，任何硬门禁失败都必须弃权。

### 原则三：状态不等于方向

状态节点只能证明某种市场结构存在，不能单独证明最终价格方向。方向判断必须经过因子聚合、历史校准和市场状态匹配。

### 原则四：代理必须显式标记

OI 急降与价格波动可以作为清算代理，但不得写成“真实清算热图”；缺少付费或权威数据时应记录缺口，不得填造替代事实。

### 原则五：历史结果不是未来授权

校准记录必须包含样本量、样本外结果、期望收益、命中率、最大不利变动、有效期和失效条件。`validated` 不是永久状态。

### 原则六：弃权是正常产品输出

数据不足、来源冲突、样本不够、市场状态不匹配或组合限制触发时，弃权代表系统正确工作，不代表功能失败。

### 原则七：执行能力最后建设

在数据可信度、历史证明、组合约束和观察期验收完成前，不建设真实下单能力。自动化不能绕过人类授权和杀开关。

## 分阶段发展路线

| 阶段 | 主要目标 | 完成标准 |
|---|---|---|
| A 状态收口 | 统一版本、状态文档和代码基线 | 状态同步、测试可复现、变更可回滚 |
| B 数据可信度 | 稳定多交易所 funding/OI 和来源健康 | 跨所覆盖、单位统一、缺口降级和熔断可验收 |
| C 状态覆盖 | 补充期权、资金流、链上与事件结构 | 每类数据有来源等级、状态定义和证据边界 |
| D 历史证明 | 提升校准严谨性 | 滚动样本外、时间净化、非重叠事件、成本压力测试通过 |
| E 组合验证 | 验证建议进入账户后的风险效果 | 相关性、波动率、滑点、风险预算和杠杆门禁通过 |
| F 观察运行 | 只观察或模拟，不下单 | 连续事件样本、异常记录、建议与结果偏差可复盘 |
| G 执行评审 | 决定是否接入模拟盘或券商 | 必须单独立项和授权，不由既有门禁自动升级 |

## 当前阶段判断（2026-07-13）

| 维度 | 当前判断 |
|---|---|
| 功能架构 | P1–P5 工程链路已形成 |
| 专项测试 | 已有计算、页面、校准、组合、执行门禁和来源可信度测试 |
| 实时来源 | 多交易所链路已建立，但来源稳定性仍需持续观察 |
| 历史证据 | 已产生部分有效校准候选，但仍需更严格的样本外验证 |
| 组合能力 | 约束模块已建立，真实账户环境尚未验证 |
| 执行能力 | 未开放，保持 advisory-only |
| 总体定位 | 研究级可用，处于证据强化和运行观察阶段 |

## 后续验收必须回答的问题

每轮迭代不只检查“文件是否存在”，还必须回答：

1. 真实数据源当前是否可达；
2. 数据单位和更新时间是否正确；
3. 跨交易所分歧是否触发硬门禁；
4. 缺失数据是否正确显示为缺失而不是零；
5. 同一因子是否发生重复计票；
6. 校准是否包含真实样本外验证；
7. 手续费、滑点和相关性是否改变结论；
8. 门禁失败时是否稳定弃权；
9. LLM、强制字段或前端操作能否绕过授权；
10. 每个结论能否回溯到原始证据。

## 指导性结论

> 微观层后续不应以“增加多少指标”作为主要进度，而应以“有多少结论能够被可靠数据支持、被历史样本验证、被组合约束审查，并在证据不足时稳定弃权”作为完成度标准。

最终价值不是告诉用户市场一定往哪里走，而是判断：

> 当前证据是否足以让账户承担更多风险；如果不足，系统能否清楚、稳定、不可绕过地选择克制。
