# 微观层功能分拆规划与实施（2026-07-13）

## 结果

微观层已从首页单一摘要区分拆为独立功能页。首页保留摘要和入口，独立页承担证据检查、状态解释、因子去重、市场状态、校准门禁、组合约束和实施路线展示。

## 页面分拆

| 模块 | 回答的问题 | 输出边界 |
|---|---|---|
| 标的总览 | 哪些币种有数据、状态和结论？ | 只做横向比较 |
| 数据与状态明细 | 原始数据经过什么归一化？ | 缺失值显示 `--`，绝不显示为 0 |
| 数据质量门禁 | 新鲜度、来源、OI 基线、周期是否合格？ | 未达阈值立即弃权 |
| 状态节点 | 当前观察到什么微观结构？ | 证明状态，不证明最终方向 |
| 因子聚合 | 多个指标是否重复表达同一件事？ | 同一因子只贡献一次 |
| 市场状态 | 趋势、波动、流动性和宏观环境是什么？ | 代理变量明确标记 |
| 仓位建议 | 是否满足计算仓位变化的条件？ | 缺校准或组合信息时不计算 |
| 校准与组合上下文 | 历史证据、账户约束是否齐全？ | 缺一项即弃权 |
| 实施路线 | 哪些已完成、哪些仍受证据门禁约束？ | 防止把规划写成完成项 |

## 本次具体实施

1. 新建独立页面 `micro.html` 和详细渲染模块 `static/micro_dashboard.js`。
2. 首页导航和原摘要区增加“微观层”详细页入口。
3. 扩充标准化结果，保留 funding、OI、24h 价格、成交额、单位和来源元数据。
4. 增加逐标的选择、KPI、质量检查条、状态/因子明细、市场状态、仓位约束和路线图。
5. 增加手动实时刷新；无缓存时，服务模式自动执行首次采集。
6. 对 `file://` 明确显示“缓存查看模式”，并提供服务版链接。
7. 修复浏览器顶层 `const Store/DataAPI/CONFIG` 无法通过 `globalThis` 取得的集成问题；此前模块可能加载但无法真正采集或读取缓存。
8. 修复 `null` 被格式化为 `0.00%` 的问题；缺失数据统一显示 `--`。

## 继续实施计划

### P1 数据口径

- 接入动态 funding 周期；完成 Binance/OKX 等跨交易所 OI 的 USD 名义价值统一；记录源级健康、延迟和降级原因。

### P2 状态覆盖

- 接入清算、期权偏度、交易所净流量、巨鲸方向、稳定币供给、长期持有者、MVRV 和代币解锁。

### P3 历史证明

- 建立“因子 × 市场状态 × 标的 × 持有期”样本；记录期望收益、命中率、最大不利变动、样本外表现与失效日期。

### P4 组合约束

- 加入账户规模、当前暴露、相关性簇、目标波动率、流动性、滑点、单资产和总杠杆限制。

### P5 执行门禁

- P1-P4 全部验收前保持 `advisory-only`；任何 LLM 输出不得绕过结构化弃权条件。
