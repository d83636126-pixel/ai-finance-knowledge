# 推送逻辑解析与用户有效性判断 · 2026-07-12

## 总判断

| 通道 | 对「研究工作台用户」有效性 | 说明 |
|------|---------------------------|------|
| ServerChan 微信 | **低 / 基本无效** | 代码在 Dashboard，产品已延后；须开着 `index.html` |
| Inbox「不推送」管线 | **中（工作流有效）** | 人主动扫描后入库，适合研究，不适合盯盘 |
| 今日研究队列 | **高（当前主价值）** | 打开页面才看见；是「拉」不是「推」 |
| Toast / 告警卡片 | **低～中** | 仅当前页反馈 |
| Cron / 邮件 / Telegram | **无** | 未实现 |

**一句话：** 系统目前是「打开再提醒」，不是「离开也告警」。对研究闭环够用；对「错过就亏」的实时告警无效。这与决策「先理解、后告警」一致，但要认清边界。

## 「推送」在本项目里被混用的四层

### 1. 外部告警（真·推送）— ServerChan → 微信

- 入口：仅 `index.html` + `Engine.init()` 定时扫描（默认约 10 分钟）
- 触发：滚动窗口综合分 `≥ 0.8` + 已配置 `key_serverchan` + 未触达日限额(5) + 标的去重窗(60 分钟)
- 内容：标的、评分、维度、一句「请前往仪表盘…」——**几乎不含传导路径 / evidence / 情景**
- **致命限制：** `propagation.html` **不加载** `engine.js` → 你日常用的研究页**根本不会发微信**
- 产品状态：P3 / V2.2 **明确延后**（见决策记录、路线图）

→ **对当前用户实际有效性：接近 0**（除非专门挂着旧 Dashboard 页）。

### 2. Inbox 摄取 — 明确「不推送」

- UI 文案：`扫描 → 选节点 → 确认分析 · 不推送`
- RSS / 症状 / 手工 / Alert 入 Inbox：均为**用户点击后**才跑
- 价值：把突发变成可分析事项；**不会**在你离开时喊你

→ **有效性：研究流程有效，告警无效。**

### 3. 今日研究队列 — 应用内「软推送」（拉模式）

打开总览时聚合：

- 日历 ±7 天生命周期（待发生 / 待事后验证…）
- Inbox 待确认 / 已映射 / 已分析
- 图谱证据过弱或失效提醒

排序后最多 8 条，带一键动作。

→ **有效性：对「今天先干什么」很高；对「人不在电脑前」为零。**  
详见 [[今日研究队列_功能说明_2026-07-12]]。

### 4. 其它叫「推送」的东西

- 归档同步 `pushToArchive`：多机同步，非告警
- 文档里「推送代码仓」：git push
- `architecture.html` 里 Cron / 9:00 简报：愿景图，**未落地**

## 触发条件对照（实际 vs 想象）

| 用户以为会推 | 实际 |
|--------------|------|
| NFP/CPI T-1 微信提醒 | ❌ 仅队列里出现「待发生」卡片（打开页才见） |
| Evidence 从无到有自动微信 | ❌ 仅队列「证据覆盖不足」提醒（打开页） |
| Inbox 突发秒推微信 | ❌ 须手工扫描；入库后不推 |
| 风险「偏紧张」推送 | ❌ 总览展示；无外发 |
| Dashboard 高分异动微信 | ⚠️ 仅旧页+密钥+标签页存活时可能 |

## 有效性拆解

### 对「宏观研究用户」（你的主画像）— **队列有效，推送可继续延后**

- 需要的是：今天看哪个事件、事后有没有做完、证据够不够  
- 队列 + Inbox + 聚焦图已经覆盖  
- 过早微信推送会把未验真路径当警报，违背「先理解、后告警」

### 对「盯盘 / 外出也要知」— **当前无效**

缺：

1. 7×24 进程（关浏览器即停）
2. 与研究工作台绑定的触发（日历邻近、高优 Inbox、关键 evidence 变化）
3. 可读载荷（路径摘要 + 情景一句 + 深链回工作台）
4. 静默时段 / 配额策略落地（配置项多半未接线）

### 对「误报成本」— **延后是对的**

旧 ServerChan 载荷偏「评分吓人」，不含路径与 evidence 分层；在图谱可信度与盲测未闭环前推送，有效性会被误报抵消。

## 建议口径（对内）

1. **对外不要说「系统会推送」** — 说「打开工作台看今日研究队列」。
2. 真要做推送，按 V2.2：**一条通道 + 三类触发（日历邻近 / 高优 Inbox / evidence 关键变化）+ 深链回 `propagation.html`**，且必须后台 worker，不要挂在 Dashboard `setInterval`。
3. 在推送上线前，把「队列第一项」当唯一主动作入口（主按钮已绑定），比半吊子微信更有效。
