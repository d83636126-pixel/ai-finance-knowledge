## 一、已经真正可用的能力

### 1. 图谱核心结构完整

当前 `propagation_graph.json` 实际包含约 **128 个节点、327 条边**。图谱健康分数为 **94**，无 health error。

非农、CPI、PPI 等核心事件均能传导至至少七个目标资产，并可抵达 BTC。核心路径来源 URL 覆盖率为 **116 / 116**。

图谱已支持：

- 正向事件推演。
- 多路径传播。
- 资产敞口计算。
- 反向归因。
- 路径权重与时滞。
- verifier 验证边。

### 2. 工作台阅读顺序合理

```text
风险状态
→ 主要驱动
→ 研究队列
→ 选择事件
→ 聚焦路径
→ Evidence
→ 报告
```

聚焦图实测约12个节点，主路径编号、左右阅读方向、上下游展开、全图切换和资产反向归因均能工作。

## 二、当前最重要的不足

### 1. Evidence 只是“可连接”，还不是“可确认”

虽然 verifier 已达到 **9 / 9**，但九项状态全部是 `partial`，没有真正的 `confirmed` 或 `invalidated`。

其中存在明显的代理指标：

- MOVE：TLT 实现波动率代理。
- Gold ETF flow：GLD 价格和成交量代理。
- Nasdaq breadth：QQQ 与 IXIC 相对表现代理。
- BTC ETF flow：来源解析值或代理值。
- Treasury auction tail：包含合成计算。

当前 Evidence 可以回答“数据大致向哪个方向变化”，但还不能稳定回答“这条传导路径是否已被真实数据确认”。

### 2. 数据新鲜度存在表达风险

`market_snapshot.json` 中部分 verifier 没有持久化 `fresh` 字段，页面总览容易展示“快照刚刷新”，但这不等于每项观测都新鲜。

例如：

- ACM 数据日期为6月30日。
- 拍卖尾部数据仍停留在5月12日。
- 部分周末市场数据来自7月10日。
- BLS 非农是月度数据。

必须明确区分：

```text
快照生成时间
≠
指标最后观测时间
```

### 3. P4 目前是结构闭环，不是真实事件闭环

自动端到端测试已通过，但当前事实是：

- 日历事件有事前报告。
- 尚无真实事后报告。
- 对比入口存在，但没有实际可比较报告。
- Inbox 当前数量为 0。
- Inbox 流程使用映射桥和默认地缘冲击节点验证。

当前证明的是“工作流接口已经接通”，尚未证明“一个真实事件从事前判断到事后复盘完整跑通”。

[[04_推演跟踪/2026-07-14_CPI-PPI-银行财报-美伊冲突_交叉窗口]] 可作为第一个真实闭环样本。

### 4. 状态源存在轻微漂移

当前图谱实际已经是约 `128节点 / 327边`，但 `project_status.json` 仍记录 `127节点 / 324边`。

这表明：

- 状态文件不是即时事实。
- 当前仓库仍是 dirty WIP。
- 自动验收状态可能滞后于实际代码。

### 5. 测试仍有“接口存在即通过”的倾向

部分端到端检查证明了对比钩子、映射桥和报告入口存在，但还不等于：

- 已生成真实事后报告。
- 已完成真实报告差异计算。
- 已处理真实 Inbox 数据。

## 三、产品定位判断

目前最适合的定位是：

> **宏观传导研究与假设管理工作台**

### 已经适合

- 分析事件可能通过什么路径影响资产。
- 找到重点中间节点。
- 组织事前假设。
- 查看部分实时证据。
- 记录支持、冲突和失效条件。
- 将研究结果沉淀到 Obsidian。

### 暂时不适合

- 自动生成确定性交易信号。
- 无人值守实时告警。
- 依赖代理指标直接判定路径成立。
- 把 `9/9 可达` 解释成 `9/9 已确认`。
- 直接用于生产级风险决策。

## 四、实际功能逐项验收

本次评价已补充逐项实测口径，详细矩阵见：[[图谱实际功能逐项验收_2026-07-12]]。

> [!warning] 2026-07-12 21:05 复验更新
> 当前版本 `check` 6/6；但 `smoke` 11/13、`acceptance` 12/14，均未全通过。失败项为 P1 可信度分层/图谱注释缺失，以及 P2“展开上下游”按钮不可见。性能基线同时因首屏 2427ms 超过 2000ms、分栏拖拽未成功而未通过。以下评价必须以此最新复验为准。

验收不再把“有按钮或有函数”视为通过，而是分为：**已实测通过、条件通过、尚未实测、待真实事件、不通过**。当前结论如下：

- 图谱核心链路（加载、事件聚焦、全图、主路径、反向归因）：通过；邻域展开当前失败。
- 工作台主链路（导航、模式、报告阅读、路径联动、持久化）：通过；可信度分层与分栏拖拽当前失败。
- 证据链（刷新、验证点、状态表达）：条件通过，9 项当前均为 `partial`，仍含代理/合成数据。
- 图形辅助控件（搜索、筛选、骨架、标签、动画、PNG、编辑器等）：实现存在，但需逐项实操补证。
- 事件复盘（post/compare）：等待真实事件后验收。
- 生产级交付：暂不通过。

## 五、后续优先级

1. 用7月14–15日真实事件完成第一次 `pre → actual → post → compare`。
2. 将快照时间与每项指标观测时间彻底分开。
3. 为 verifier 建立“官方原指标 / 合理代理 / 手工数据”三级标签。
4. 至少让一个真实路径产生 `confirmed` 或 `invalidated`。
5. 用真实 Inbox 条目重跑突发事件闭环。
6. 更新 `project_status.json`，提交当前 WIP，再重跑全套验收。

## 最终判断

> [!summary]
> 图谱主体已经具备较强研究价值，交互和性能基本成熟；目前短板集中在证据质量和真实事件闭环，而不在图谱展示本身。
