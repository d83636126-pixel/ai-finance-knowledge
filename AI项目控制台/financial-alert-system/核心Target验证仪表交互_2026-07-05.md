# 核心 Target 验证仪表交互

## 背景

上一轮已经在路径报告和 FOMC Case 报告顶部加入“核心 target 验证仪表”。

本轮继续把仪表从静态展示升级为可交互：

```text
点击 BTC / Nasdaq / Gold / USD / Treasury 卡片
展开该资产自己的验证详情和相关路径
```

## 本次实现

已更新：

```text
propagation.html
```

新增函数：

```text
targetPanelId(scope, id)
showTargetValidationDetail(scope, targetId)
```

增强函数：

```text
renderTargetValidationDashboard(targets, paths)
```

## 展示效果

核心 target 卡片现在可以点击。

点击后：

- 当前卡片高亮。
- 隐藏其他 target 的详情。
- 展开当前 target 的验证详情。
- 展示最多 5 个验证指标。
- 展示每个指标的观察说明。
- 展示对应失效条件。
- 如果报告里有该 target 的路径，展示相关路径。

## 接入位置

- FOMC Case 报告。
- 普通路径报告。

## 后续建议

下一步建议做浏览器人工验收：

- 点击 BTC / Nasdaq / Gold / USD / Treasury 是否顺滑。
- 展开的验证详情是否太长。
- 失效条件是否需要压缩成更短的交易判断语言。

之后可以继续做：

- 点击验证指标时，在图谱中定位对应 verifier 节点。
- 点击相关路径时，高亮该路径。
- 给核心 target 仪表增加“确认 / 待确认 / 失效”状态色。
