# financial-alert-system：传播图研究工作台整改方案

> **进度（2026-07-11）**
> - P0–P4 **全部完成**
> - P4：报告摘要/全文折叠、段落反高亮路径、比例/导航记忆、窄屏藏图、Alt+1…6 / Esc / skip-link
> - 冒烟：`smoke_research_workbench_p1`…`p4` 均 PASS

## 一、问题定义

当前 `propagation.html` 的第一视觉中心是完整传播图，但用户打开页面后缺少明确的阅读顺序：

- 第一眼看到大量节点，却不知道应该先看哪个事件、资产或路径。
- 图谱占据主要页面面积，但完整图谱更适合探索和维护，不适合作为默认阅读入口。
- 右侧仪表板承载结论、数据佐证、日历、Inbox、推演和报告等关键能力，空间比例过小。
- 面板内容多而密，文字阅读费劲，长报告和验证详情尤其不舒服。
- 日历、Inbox、Case、推演和 evidence 功能虽然完整，但缺少统一的任务入口与下一步指引。

### 核心矛盾

```text
当前页面主角 = 完整图谱
用户真正需要 = 当前状态 + 建议先看 + 下一步操作
```

整改目标不是削弱图谱，而是把图谱从“默认展示对象”调整为“解释当前问题的核心工具”。

## 二、产品定位

将传播图页面从“图谱展示页”调整为：

> **宏观传导研究工作台**

各模块职责：

| 模块 | 负责回答 |
|---|---|
| 研究工作台 | 当前发生了什么、先看什么、下一步做什么 |
| 日历 | 未来和最近有哪些计划内事件 |
| Inbox | 有哪些突发事件等待确认和分析 |
| 推演 | 事件通过什么路径影响核心资产 |
| Evidence | 真实数据是否支持或否定推演 |
| 图谱 | 为什么会产生这些影响，路径如何连接 |
| 报告 | 事前判断、事后结果和差异是什么 |

## 三、设计原则

1. **结论先行**：先显示当前状态、重点事件和资产影响，再显示图谱细节。
2. **任务驱动**：默认告诉用户下一步应该看哪里，而不是让用户自己探索全部节点。
3. **图谱聚焦**：默认只显示与当前事件或资产相关的主路径。
4. **渐进展开**：摘要 → 主路径 → evidence → 完整报告 → 全图探索。
5. **保持主线**：图谱仍是推理核心，但不再成为首次进入页面的理解门槛。
6. **复用现有能力**：不重新实现日历、Inbox、Case、ScenarioReasoner 和传播引擎。
7. **证据分层**：理论、部分确认、确认、失效、无数据必须明确区分。

## 四、目标页面结构

### 4.1 桌面端默认布局

```text
┌──────────────────────────────────────────────────────────────┐
│ 全球风险状态 · 数据更新时间 · 系统状态 · 页面模式             │
├────────────────────────────┬─────────────────────────────────┤
│                            │                                 │
│     聚焦后的传导图谱        │          研究工作台             │
│          55%               │             45%                 │
│                            │                                 │
│  仅显示当前事件相关主路径   │ 状态 / 队列 / 资产 / 证据 / 操作 │
│                            │                                 │
├────────────────────────────┴─────────────────────────────────┤
│ 总览 · 事件（日历/Inbox）· 推演 · 验证 · 报告 · 全图           │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 面板比例

- 默认：图谱 55% / 工作台 45%。
- 阅读报告：图谱 30% / 工作台 70%。
- 全图探索：图谱 80% / 工具面板 20%。
- 分隔线支持拖动，建议约束在 30%–80% 范围内。
- 记住用户上次选择的比例和模式。

### 4.3 响应式布局

- 宽屏：图谱与工作台左右并排。
- 窄屏：工作台优先，图谱以独立页签或抽屉打开。
- 手机端不加载完整全图作为首屏。

## 五、首次打开页面

默认进入“今日总览”，不直接展示无焦点的完整图谱。

### 5.1 全球风险状态

第一屏只展示：

- 当前全球风险状态及方向。
- 最多三个主要驱动因素。
- BTC、Nasdaq、Gold、USD、Treasury 的方向摘要。
- evidence 确认、部分确认、失效、无数据的数量。
- 数据最后更新时间。

示例：

```text
全球风险状态：偏紧张

主要驱动
1. 实际利率上升
2. 美元压力增强
3. 风险偏好下降

资产影响
BTC ↓  Nasdaq ↓  Gold →  USD ↑  Treasury ↓

证据：5 项确认 · 2 项部分确认 · 2 项暂无数据
```

主操作只保留：

> **查看当前传导路径**

点击后图谱聚焦到对应路径。

### 5.2 今日研究队列

工作台自动生成建议先看的项目：

1. 即将发生的重要日历事件。
2. 已公布但尚未完成事后对比的事件。
3. Inbox 中尚未确认映射的突发事件。
4. evidence 已失效或数据过期的推演。
5. 核心资产中发生显著方向变化的项目。

每条队列卡必须包含：

- 事项名称。
- 为什么值得看。
- 当前生命周期状态。
- 一个明确的下一步按钮。

示例：

```text
CPI 公布后待验证
原因：实际值高于预期，可能影响实际利率与美元
[查看事后对比]
```

## 六、三种页面模式

### 6.1 今日总览（默认）

适用：日常打开页面快速判断当前重点。

包含：

- 全球风险状态。
- 今日研究队列。
- 核心资产篮子。
- evidence 新鲜度。
- 最近报告。

### 6.2 聚焦研究

适用：用户已选择一个事件、资产或 Case。

包含：

- 当前研究对象与生命周期状态。
- 聚焦后的主路径图谱。
- 资产敞口。
- 支持证据、冲突证据和失效条件。
- 事前、事后和对比报告入口。

### 6.3 全图探索

适用：图谱维护、节点搜索、逆向归因和深度探索。

包含：

- 完整图谱。
- 节点类型与区域筛选。
- 搜索和反向归因。
- 节点详情。

全图探索不是默认模式。

## 七、图谱整改

### 7.1 默认可见范围

默认聚焦图只显示：

```text
当前事件
  → 关键中间节点
  → 全球风险状态
  → 核心资产
  → 关键 verifier
```

- 建议控制在 15–25 个可见节点。
- 无关节点默认隐藏，而不是仅降低透明度。
- 用户可点击“显示完整上下游”继续展开。

### 7.2 固定阅读方向

默认采用从左到右布局：

```text
事件 → 宏观状态 → 全球风险状态 → 资产 → 验证指标
```

### 7.3 主路径编号

主路径按照阅读顺序编号，例如：

```text
① CPI 偏热
② 降息预期下降
③ 实际利率上升
④ 全球风险状态收紧
⑤ Nasdaq / BTC 承压
```

### 7.4 视觉层级

- 主路径：高对比色、粗线、完整标签。
- 次要路径：降低透明度。
- 冲突证据：黄色或橙色。
- 已失效路径：虚线并显示失效标记。
- 暂无数据：中性灰色，不伪装成确认。
- 当前选中事件和核心资产始终保持视觉锚点。

### 7.5 工作台与图谱联动

| 工作台操作 | 图谱反应 |
|---|---|
| 点击日历事件 | 显示该事件的主路径 |
| 点击 Inbox 条目 | 显示候选节点和待确认映射 |
| 点击核心资产 | 反向显示影响该资产的主要路径 |
| 点击 evidence | 定位对应 verifier 和验证边 |
| 点击报告结论 | 高亮支撑该结论的路径 |
| 点击失效条件 | 高亮可能使推演失效的节点 |

## 八、功能导航重组

将当前零散功能整理为六个一级入口：

| 一级入口 | 内容 |
|---|---|
| 总览 | 全球风险、研究队列、资产影响、最近报告 |
| 事件 | 日历 + Inbox；计划内与突发事件统一管理 |
| 推演 | 当前路径、Scenario、Case |
| 验证 | Market、verifier、evidence、失效条件 |
| 报告 | 事前、事后、对比、历史结果 |
| 全图 | 完整图谱探索和维护 |

日历和 Inbox 不删除，只是在导航层合并为“事件”的两个子页签。

## 九、事件生命周期

所有日历和 Inbox 事件统一展示当前阶段：

```text
待发生
  → 已生成事前报告
  → 已公布 / 已发生
  → 待事后验证
  → 已完成对比
```

突发事件使用：

```text
待确认
  → 已映射图谱
  → 已完成即时分析
  → 待事后验证
  → 已完成对比
```

生命周期状态用于生成今日研究队列。

## 十、报告阅读整改

打开报告时，工作台自动扩大到 65%–70%。

报告信息顺序固定为：

1. 一句话结论。
2. 核心资产影响。
3. 主传导路径。
4. 支持证据。
5. 冲突证据。
6. 失效条件。
7. 下一步观察项。

长报告默认展示摘要，可展开完整正文。报告段落应能够反向高亮图谱路径。

## 十二、第一版验收标准

### 首屏理解

- [ ] 新用户打开页面后，无需操作即可知道当前风险状态。
- [ ] 首屏明确展示“建议先看”的事项及下一步按钮。
- [ ] 首屏不会直接展示无焦点的完整 124 节点图谱。

### 页面比例

- [ ] 工作台默认宽度不低于页面的 40%。
- [ ] 用户可以拖动调整图谱与工作台比例。
- [ ] 长报告模式下工作台宽度达到 65%–70%。

### 图谱可读性

- [ ] 聚焦模式默认可见节点控制在约 15–25 个。
- [ ] 主路径有明确阅读方向和编号。
- [ ] 无关节点默认隐藏。
- [ ] 支持从工作台直接定位对应路径、资产和 verifier。

### 工作流

- [ ] 日历事件可进入聚焦推演和事前/事后报告。
- [ ] Inbox 条目可完成映射确认并进入聚焦推演。
- [ ] 报告可以返回事件和图谱路径。
- [ ] 现有 Case、Alert、Market 和 evidence 功能不回退。

### 性能

- [ ] 今日总览不依赖完整图谱全部渲染完成后才能阅读。
- [ ] 模式切换和分栏调整不会重复加载整份数据。
- [ ] 聚焦图的交互响应明显快于完整图谱。

## 十三、明确不做

- 不重新开发一套传播引擎。
- 不删除完整图谱探索能力。
- 不把所有功能同时铺在首屏。
- 不在仪表板复制完整日历和完整 Inbox。
- 不增加大量与研究任务无关的行情装饰卡片。
- 不把 AI 摘要显示为无证据的确定性结论。
- 不在源码分叉未解决前直接修改 Downloads 并继续扩大分叉。

## 十四、最终完成定义

整改完成后，页面的默认使用顺序应变为：

```text
第一眼看状态
  → 第二步看建议
  → 第三步选择事件或资产
  → 图谱解释主路径
  → evidence 验证
  → 事后对比并沉淀报告
```

最终判断标准：

> 用户打开页面后，不需要先理解完整图谱，也能在十秒内知道“现在发生了什么、为什么重要、下一步点哪里”。
