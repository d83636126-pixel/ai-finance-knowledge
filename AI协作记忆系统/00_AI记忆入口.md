# AI记忆入口

> 这份笔记是 Codex / Claude / 其他 AI 进入本 vault 后应优先读取的入口。

## 第一属性：长期记忆

Obsidian 在这里不是普通资料夹，而是长期记忆层。

它要解决的问题：

- AI 单次会话会遗忘，vault 负责保存长期上下文。
- 项目会跨天、跨工具、跨模型推进，vault 负责保存连续性。
- 人负责判断方向是否 OK，AI 负责把对话、行动、结论整理成结构化笔记。
- GitHub 负责迁移和同步，换机器、换工具后仍然能恢复上下文。

## 第二属性：项目记录

每个重要项目都应该有一个“项目主文档”，主文档下面再分出相关记录。

推荐结构：

```text
项目主文档
├─ 当前状态
├─ 目标与边界
├─ 决策记录
├─ 任务进度
├─ 会话交接
├─ 资料与证据
└─ 下一步行动
```

AI 的职责不是只回答问题，而是一边协助、一边整理项目树。

## AI工作规则

每次开始处理本 vault 相关任务时：

1. 先读本文件。
2. 再读 [[AI协作记忆系统/项目_Codex与Claude协作记忆系统]]。
3. 如果是延续任务，再读 [[AI协作记忆系统/会话交接日志]]。
4. 执行任务时，把关键判断、节点、进度写回相关笔记。
5. 结束前更新“当前状态”和“下一步行动”。
