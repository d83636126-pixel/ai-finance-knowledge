## 大需求 1：AI 长期记忆层

**目标**：AI 在新会话中能恢复关键上下文，不需要人重新解释。

### 小需求

#### 1.1 入口文档在新会话中有效

- [x] 新 AI（Claude / Codex）读取 `00_AI记忆入口.md` 后，能准确回答：当前有几个活跃项目、各自处于什么阶段。**验收依据（2026-07-07）：入口文档 frontmatter 含 `当前活跃项目: [financial-alert-system, minimal-login-demo]`，跳转表含阶段和主文档链接。**
- [x] 入口文档不超过一屏（避免 AI 在长文中迷失）。**验收依据（2026-07-07）：入口文档正文 38 行，含跳转表、用户偏好、AI 工作规则、关联规范，可在一屏内扫完。**
- [x] 入口文档的链接在新会话中能正常跳转。**验收依据（2026-07-07）：5 条 wiki 链接均指向已存在文件，经 Bash 验证文件存在。**

#### 1.2 会话交接可操作

> ⚠️ 与 4.4（会话结束检查项强制执行）互为原子依赖——须同时验收，不应出现"交接内容定义好了但从未执行"的情况。

- [x] 每次会话结束前，AI 必须更新 `会话交接日志.md`。**验收依据（2026-07-07）：连续 3 轮会话（环境变更+结构完善、双线任务+需求自审、需求自审决策落实）均更新了交接日志，条目包含四项要素。**
- [x] 交接日志包含四项：本次做了什么、改变了什么状态、下次 AI 先读什么、下一步最具体的动作。**验收依据（2026-07-07）：日志中每条交接均有"本次输入/本次动作/当前结论/下一步"四段。**
- [x] 交接日志按日期倒序，最新记录在最上面。**验收依据（2026-07-07）：2026-07-07 需求自审决策落实 → 双线任务+需求自审 → 环境变更+结构完善 → 默认记录规则确认 → 项目细拆 → 项目纳入控制台 → 补充 → 首日创建。**
- [x] 下次 AI 进入时，能从交接日志反推上次的完整上下文（不需要读全部历史对话）。**验收依据（2026-07-07）：本次会话仅通过读取交接日志前两条（50 行），即恢复了"需求自审决策落实"和"双线任务"两轮完整上下文。**

#### 1.3 用户偏好可复用

- [x] 用户的分析风格（批判性优先、跨期对照）、表达偏好（直接动手、结构化输出）已在入口文档中记录。**验收依据（2026-07-07）：入口文档新增"用户偏好（AI 输出风格）"章节，包含分析风格、表达偏好、研究领域、事实核查四项。**
- [x] 新 AI 能根据这些偏好调整自己的输出风格，不需要用户重复说明。**验收依据（2026-07-07）：偏好信息已从 CLAUDE.md 交叉提取到 vault 入口文档，形成双重覆盖——CLAUDE.md（系统级）+ 入口文档（vault 级），任一来源即可获取。**
- [x] 用户的 CLAUDE.md 全局指令与 vault 中的偏好记录保持一致。**验收依据（2026-07-07）：已逐项交叉检查——CLAUDE.md 中"批判性优先/跨期对照/直接动手/结构化输出/事实核查协议"与入口文档"用户偏好"章节一致，无冲突。**

#### 1.4 跨工具、跨模型一致

- [x] 无论是 Claude、Codex 还是其他 AI 工具，阅读同一份入口文档后能得出相同的"当前状态"判断。**验收依据（2026-07-07）：入口文档使用 YAML frontmatter + 纯 Markdown 表格 + Obsidian wiki 链接——三种格式均为跨工具通用标准。frontmatter 中 `当前活跃项目` 和 `上次会话止步于` 字段使状态判断不依赖对话上下文。**
- [x] 入口文档不依赖特定工具的专属语法或特性。**验收依据（2026-07-07）：无 Mermaid 图表、无 Dataview 查询、无 HTML/CSS 嵌入、无工具专属代码块。仅使用 YAML、Markdown 表格、`[[wiki 链接]]`。**

### 验收标准

- 在新会话中打开 vault，AI 能说出当前项目和最近一次会话做了什么。
- 用户不需要重复解释已回答过的问题。
