# AI 协作记忆系统 —— 执行规划

> 本文件基于 `需求拆分.md`（26 条小需求）和 `需求自审摘要.md`（自审结论 + 已落实决策）整理，
> 供 Claude Code 按阶段、按优先级逐条执行。执行时请勾选 checkbox，并按"每阶段收尾动作"回写 vault。

## 给 Claude Code 的执行说明

1. 开始前，先读 `00_AI记忆入口.md` 和 `会话交接日志.md`，确认当前 vault 状态，不要假设本文件是唯一信息源。
2. 按下面的阶段顺序执行：P0 → P1 → P2 → P3。同一阶段内条目无强制顺序，但 1.2 与 4.4 须同时验收（原子依赖）。
3. 每完成一条小需求，在 `需求拆分.md` 对应位置把 `[ ]` 改成 `[x]`，并追加一句验收依据（参考文件中已有条目的写法）。
4. 涉及路径/环境判断时，严格遵守 2.6 的规则：路径不可访问 ≠ 源码丢失，先判断是否为多机协作场景。
5. 每个阶段做完后执行"阶段收尾动作"（见文末），不要跳过会话交接日志的更新。
6. 遇到本文件与 `需求拆分.md` 内容不一致的情况，以 `需求拆分.md` 最新版本为准，并在交接日志中记下差异原因。
