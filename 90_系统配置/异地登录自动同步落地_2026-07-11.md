# 异地登录：自动同步落地指南

> 关联：[[90_系统配置/多机器同步Obsidian库指南]] · [[AI项目控制台/financial-alert-system/_sync/README]]

## 一句话

异地登录**不要依赖固定盘符**（F: / D:）。  
以 **GitHub 为唯一中转**；每台机器本地各自 clone；打开工作前 **pull**，写完 **push**；涉及验收状态再跑 **status sync**。

```text
机器 A 改代码/笔记
  → push 到 GitHub
机器 B 异地登录
  → pull 文档库 + 代码库
  → 跑 sync_vault_status（用本机实际 code_root）
  → Obsidian AUTO 区块与本机事实对齐
```

## 两个仓库（都必须）

| 仓库 | GitHub | 本机建议路径（可改） |
|---|---|---|
| Obsidian 文档库 | `https://github.com/d83636126-pixel/ai-finance-knowledge` | `D:\AI  金融知识点` 或任意目录 |
| 代码仓 | `https://github.com/d83636126-pixel/financial-alert-system` | `D:\financial-alert-system` |

文档里的 `F:\...` 只是**某台历史机器路径**，异地机器没有 F: 盘是正常的。

## 新机器首次落地（一次）

在 PowerShell：

```powershell
# 1) 文档库
git clone https://github.com/d83636126-pixel/ai-finance-knowledge.git "D:\AI  金融知识点"

# 2) 代码库
git clone https://github.com/d83636126-pixel/financial-alert-system.git "D:\financial-alert-system"

# 3) 用本机路径跑状态同步（校正验收 AUTO 区块）
powershell -NoProfile -ExecutionPolicy Bypass -File "D:\AI  金融知识点\AI项目控制台\financial-alert-system\_sync\sync_vault_status.ps1"
```

或直接跑引导脚本（库内已提供）：

```powershell
powershell -File "D:\AI  金融知识点\90_系统配置\bootstrap_remote_machine.ps1"
```

然后：

1. Obsidian → Open folder as vault → 选文档库目录  
2. 安装插件 **Obsidian Git**（推荐）  
3. Cursor 打开文档库（默认工作区）

## 日常异地仪式（每次换机器）

### 开始前（必须）

```powershell
# 文档
cd "D:\AI  金融知识点"   # 换成你的实际路径
git pull --ff-only

# 代码
cd "D:\financial-alert-system"
git pull --ff-only

# 用本机代码事实刷新 Obsidian 验收状态
powershell -File "...\AI项目控制台\financial-alert-system\_sync\run_sync.bat"
```

### 结束后（必须）

```powershell
# 若改了代码
cd "D:\financial-alert-system"
git add -A
git commit -m "feat: ..."
git push

# 再同步状态进文档，并推文档库
cd "D:\AI  金融知识点"
powershell -File ".\AI项目控制台\financial-alert-system\_sync\run_sync.bat"
git add -A
git commit -m "docs: sync status from code probes"
git push
```

口诀：

```text
异地开机：双 pull → sync
异地收工：代码 push → sync → 文档 push
```

## 自动到什么程度

| 层级 | 做什么 | 是否已有 |
|---|---|---|
| L1 内容同步 | GitHub pull/push 文档+代码 | ✅ 仓库已有 |
| L2 状态对齐 | `sync_vault_status` 探针回写 AUTO 区块 | ✅ `_sync/` |
| L3 Obsidian 内一键 | Obsidian Git 插件定时 pull/push | 推荐安装 |
| L4 无人值守 | 计划任务 / Cursor Automation 定时跑 sync+push | 可选增强 |

**异地最小可用 = L1 + L2。**  
没有 L2，就会再次出现「代码已做完、Obsidian 还写未验收」。

## Obsidian Git 插件建议

- Vault：文档库  
- Pull on startup：开  
- Auto commit/push：建议**先关**或仅在单机使用时开；异地多机易冲突  
- 手动：开始工作 Pull，结束工作 Commit+Push  

## 路径登记（避免再写死 F:）

`_sync/acceptance_registry.json` 已支持多候选：

```json
"code_root_candidates": [
  "D:\\financial-alert-system",
  "F:\\financial-alert-system",
  "C:\\Users\\Administrator\\Downloads\\financial-alert-system"
]
```

新机器若路径不同：把本机 code 路径**加到列表最前面**，再跑 sync。

## 冲突与脏工作区

- 两台同时改同一笔记 → 后 push 的会冲突；习惯「开始前 pull」可大幅避免  
- sync 只会改 `AUTO:STATUS` 区块和部分 `status:` frontmatter，一般不与正文冲突  
- 若文档库有本地未提交改动又要 pull：先 `git stash` → pull → stash pop → sync

## 验收状态在异地如何才算数

1. 代码变更必须已 **push 到 GitHub**  
2. 异地机器 **pull 到代码**  
3. 跑 **sync**（探针看到本机文件）  
4. 再 **push 文档库**（AUTO 区块带上新状态）

只改 Obsidian 勾选、不 push 代码 = 无效。  
只 push 代码、异地不 sync = Obsidian 仍滞后。

## 明确不做

- 不靠网盘（坚果云/OneDrive）双写同一 Git 库  
- 不假设所有机器都有 F:  
- 不把 Downloads 当正式源码根长期使用  
- 不让聊天记忆充当跨机器状态源
