# 文件夹 / 文件状态维护手册（方案 D）

**编辑入口**

- 文件夹：[[folder_status.yml]]
- 文件覆盖（与夹色不同才写）：[[file_status.yml]]
- 用户说明：[[00_状态色标说明]]
- 自动总览：[[00_状态总览]]

**知识库根路径（以本机为准）**

```text
F:\AI 金融知识点
```

## 命令

```powershell
# 只检查（默认）
powershell -NoProfile -ExecutionPolicy Bypass -File "F:\AI 金融知识点\90_系统配置\sync_folder_status_colors.ps1" -Check

# 预览变更（不写盘）
powershell -NoProfile -ExecutionPolicy Bypass -File "F:\AI 金融知识点\90_系统配置\sync_folder_status_colors.ps1" -Preview -Dashboard

# 正式同步 + 重建总览
powershell -NoProfile -ExecutionPolicy Bypass -File "F:\AI 金融知识点\90_系统配置\sync_folder_status_colors.ps1" -Sync -Dashboard
```

效果（`-Sync`）：

- 按计算颜色更新文件夹 + 继承文件的侧栏色条（CSS）
- 仅索引改名为 `00_<emoji>索引.md`，并修补相关双链
- 可选重建 `00_状态总览.md`
- **不会**给普通文件加 emoji 文件名前缀

## 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| `path` | 是 | 相对库根的路径 |
| `workflow` | 是 | backlog / scheduled / active / waiting / paused / done / archived |
| `attention` | 是 | normal / watch / blocked（过期由脚本标 overdue） |
| `updated` | 是 | 最近确认日期 `yyyy-MM-dd` |
| `review_by` | 活跃类必填 | 下次复核日 |
| `next_action` | 活跃类必填 | 下一步具体行动 |
| `reason` | 橙/红必填 | 关注或阻塞原因 |
| `owner` | 项目类建议 | 维护责任方 |
| `inherit` | 可选 | 子文件是否继承夹色（默认 true） |

兼容：仍可写旧字段 `status: green|blue|orange|red|gray`，脚本会映射到 workflow/attention。

---

## 颜色计算优先级

1. `attention: blocked` → 红  
2. 超过 `review_by` → 红  
3. `attention: watch` → 橙  
4. `workflow: scheduled|waiting` → 橙  
5. `workflow: active` → 蓝  
6. `workflow: done` → 绿  
7. `paused|archived|backlog` → 灰  

---

## 状态转换

```text
backlog → scheduled → active → done
                     ↓
                  waiting
                     ↓
                   active
```

- 任意阶段可进 `paused`；恢复后回到原阶段  
- `done` 长期不维护 → `archived`  
- `blocked` 只改 attention，不自动改 workflow  
- 人工确认后必须更新 `updated`

---

## 故障处理

| 现象 | 处理 |
|---|---|
| Check 报路径不存在 | 修正 YAML 中 `path`，或先建文件夹/文件 |
| 报缺少 next_action / review_by | 活跃、排期、等待项补齐字段 |
| 报缺少 reason | 橙/红项必须写原因 |
| 索引重命名冲突 | 手动处理重复的 `00_*索引.md` 后再 Sync |
| 侧栏颜色未变 | Obsidian → 外观 → 启用 `vault-folder-status-colors`，再重载 |
| 第二次 Sync 仍有变更 | 查看输出 Changes；通常是总览/CSS 内容漂移，核对 YAML |

## 配置示例

```yaml
- path: AI项目控制台/financial-alert-system
  workflow: active
  attention: watch
  owner: AI协作
  updated: 2026-07-13
  review_by: 2026-07-20
  next_action: 完成证据层 V2 验收
  reason: 当前证据快照仍以手工数据为主
  inherit: true
```
