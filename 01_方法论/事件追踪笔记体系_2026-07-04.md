## 一、事件追踪 vs 月度跟踪

### 两个体系的分工

| 维度 | 月度跟踪 | 事件追踪 |
|------|---------|---------|
| **对象** | 非农数据 | 所有其他重大事件 |
| **频率** | 月度 | 事件驱动 |
| **触发** | 第一个周五 | 事件发生时 |
| **粒度** | 单月内 | 单事件 |
| **方法论** | 注水识别 | 5步情景推演法 v2 |

### 互补关系

```
月度跟踪（非农）
   ↓ 提供"基础面"判断
事件追踪（CPI/FOMC/地缘）
   ↓ 提供"事件冲击"分析
投资决策（综合）
```

## 七、事件追踪工作流

### 事件发生前（T-7）

```
1. 套用【事件追踪模板】
2. 填写"事件前市场预期"
3. 应用【5步情景推演法 v2】
4. 设定情景概率
5. 制定应对预案
```

### 事件发生时（T+0）

```
1. 实时记录"事件性质"
2. 比对预期vs实际
3. 评估"市场反应"
4. 应用"利益方映射"
5. 快速决策（基于预案）
```

### 事件发生后（T+1~7）

```
1. 完成"市场反应"详细记录
2. 应用"叙事话术预测"验证
3. 评估"投资决策"执行
4. 记录"关键洞察"
```

### 事件后跟踪（T+30）

```
1. 回填"跨事件关联"
2. 验证"方法论准确度"
3. 记录"成功/失败"
4. 更新"方法论改进"
```
