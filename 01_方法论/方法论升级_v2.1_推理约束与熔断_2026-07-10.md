# 方法论升级 v2.1：推理约束、压力指数与熔断

> 整理日期：2026年7月10日  
> 触发：用 v2 重评 [[深度评估_2024年12月非农_方法论v2_2026-07-10]]、[[深度评估_2025年1月非农_方法论v2_2026-07-10]] 后的元评价  
> 父文档：[[01_方法论/深度对话_6月非农后深谈方法论_2026-07-04]]  
> 原则：**补丁有上限；熔断优先于加固。**

---

## 一句话定位

v2 的价值是 **约束错误类型（guardrails）**，不是提高估计精度。  
任何看起来像概率、注水量、置信度的输出，默认视为 **表达不确定性的修辞装置**，除非通过事后校准。

---

## 〇、核心张力（必须写进首页）

| 表象 | 实质 |
|------|------|
| 格子多、拆分细 | **禁止某些推理路径**（不准把动机当交易、不准把噪音当操纵） |
| 百分数并列 | 易制造「量化过的假精度」 |
| 强项与弱项 | **同一枚硬币**：拆分防误读 ↔ 拆分不产生新信息 |

**信息增益主要来自负面清单，不来自正面清单上的带百分号格子。**

---

## 一、非信息增益声明

### 哪些是约束（保留）

| 机制 | 合法用途 |
|------|----------|
| 水平 / 变化率拆分 | 禁止「失业率低 = 就业全面强」 |
| 噪音 / 操纵双轨 | 禁止「修订大 = 有人动手脚」 |
| 四类结论拆分 | 禁止「因为注水所以立刻空仓」合并判断 |
| 叙事话术库 | 可回测的表态预测，不是仓位公式 |

### 哪些禁止当概率用

| 输出 | 正确读法 | 禁止读法 |
|------|----------|----------|
| 数据真实性 xx% | 主观确信序数的修辞 | 后验概率 / 胜率 |
| 注水量 +N 万 | 结构异常的量级直觉 | 可复现估计量 |
| 动机分析 xx% | 「有/无证据」的强度标签 | 阴谋论强度计 |
| 交易含义 xx% | **序数警示灯**（高/中/低） | 可配仓概率 |

### 修补8 升级读法

| 原设计 | v2.1 |
|--------|------|
| 四类独立打分，防合并 | **拆分仍保留** |
| 四个百分数并列 | 后三项 **不得被读成可配仓概率** |
| 交易含义可有自己的 % | 交易栏改为：**序数警示（高/中/低）+ 绑定失效条件**；若仍写 %，强制 \(c_4 \le \min(c_1,c_2,c_3)\) 并标注「非概率」 |

粗误差传播直觉（正相关时更糟）：

\[
c_{\text{trade}} \lesssim \min(c_1,c_2,c_3)
\quad\text{或更严}\quad
c_{\text{trade}} \lesssim c_1 \cdot c_2 \cdot c_3
\]

例：注水 65% × 真实性 78% × 动机 82% 量级下，交易栏若仍标 60%，**偏乐观**，不是偏保守。

## 二、样本量自我限制（反过拟合）

| 说法 | v2.1 定性 |
|------|-----------|
| 红旗阈值（政府>25%、CR3>85% 等） | **启发式门禁**，不是估计量 |
| 「按拜登/特朗普/基准月分层校准」 | **禁止称校准**；最多叫 **先验倾斜（prior tilt）** |
| 用 2026-05/06 倒推阈值再叠体制维度 | 自由度↑、独立观测几乎不↑ → **换皮过拟合** |

硬约束写进文档：

> **样本量不支持分层校准；任何分期权重不得反向改写历史案例的定性结论。**

## 三、第三轨：压力指数（不进操纵轨）

单月「无证据」丢弃红旗滚动密度会浪费信息；但把「连续三月红旗升高」自动升格为操纵，会拆掉修补6–7。

| 轨道 | 输出 | 禁止解读为 |
|------|------|------------|
| A 统计噪音 | 质量 / 修订疑点 | 操纵 |
| B 蓄意操纵 | 有 / 无具体证据 | 「红旗多 = 有人动手」 |
| **C 压力指数（新）** | 红旗数、政府占比、CR3、上修幅度的滚动分位 | **有罪认定** |

**C 的合法用途：** 提高对下月数据质量的先验警惕、加大置信度衰减、缩短持仓 horizon。  
**C 的非法用途：** 坐实操纵、替代证据门槛。

### 滚动记录字段（月度回填）

| 月份 | 红旗命中 | 政府占比 | CR3 | 近月净修订 | 压力分位（主观） | 操纵证据 |
|------|----------|----------|-----|------------|------------------|----------|
| 2024-12 | 1–2 | 12.9% | ~52% | 净 -8k | 低 | 无 |
| 2025-01 | 2–3 | 22.4% | ~70% | 近月 +100k；基准 -589k | 中（质量压力） | 无 |
| 2025-02 | 待填 | | | | | |

## 四、熔断与证伪路径

加固型修补（修订拆分、回填、衰减表）让框架活得更久；**没有「什么条件下宣布死亡」是治理缺陷。**

| 熔断触发 | 含义 | 动作 |
|----------|------|------|
| 连续 **4** 次主情景被事后归为「完全证伪」 | 情景层失效 | 暂停概率输出，只保留红旗 + 双轨（+压力指数） |
| 注水区间中位数与后续基准/两年修订的**方向**连续 **3** 次相反 | 注水层失效 | 冻结注水量数字，改输出「结构异常清单」 |
| 交易警示按「可执行」回测，样本外信息比 ≤ 0（或校准曲线系统性偏离）达 **2** 个季度 | 决策层失效 | 框架降级为「阅读清单」，**禁止进仓位规则** |
| 为消化证伪而新增修补 ≥ **2** 类且仍无法通过回填 | 补丁成瘾 | **推翻重来**，禁止 v2.x 继续叠 |

原则：

> **补丁有上限；熔断优先于加固。**  
> 熔断条款与修补清单同级，甚至更高。

---

## 五、对既有修补的处置清单

| 项 | 动作 |
|----|------|
| 修补1–7 | 保留；用途仍是约束 |
| 修补8 | **降级读法**（见第一节）；交易栏改序数警示 |
| 「体制分期红旗」建议 | **降为 prior tilt**，禁止写进校准叙事 |
| 注水量公式化 | 仍缺可复现估计；在熔断触发前只标「直觉区间」 |
| 新修补9+ | **默认禁止**；优先走熔断，不优先打补丁 |

## 六、实战回填约定

后续每份「深度评估_方法论v2」笔记应额外包含：

1. **非概率标签**：四类置信度表下加一句「非校准概率；交易栏为序数警示」  
2. **压力指数一行**：本月红旗数相对近 3–6 月的方向（升/平/降），不写操纵结论  
3. **上期情景回填**：主情景是否证伪（完全 / 部分 / 未证伪）——供熔断计数器使用  
4. **水平修订 vs 变化率修订拆分**（基准月强制）：避免「上修=更强」「下修=造假」单线叙事
