## 四层分工（方案 C）

1. **文件夹色条**：CSS（左侧目录树）
2. **索引文件名**：仅 `00_<emoji>索引.md` 带 emoji（夹级一眼可见）
3. **细分文件侧栏**：CSS 着色  
   - 默认：**继承所在文件夹色**（`data-path^="夹/"`）  
   - 例外：写入 `file_status.yml` 覆盖（如蓝夹里的稳态决策记录 → 绿）
4. **文内未完成项**：行首 🔵/🟠/🔴/⚪ 或 callout（与文件名无关）

**禁止**再给细分文件改名为 `🔵_需求拆分.md` 这类前缀（易坏链接）。

## 变更流程

1. 改夹色 → 只改 `folder_status.yml`
2. 某文件与夹色不同 → 写入 `file_status.yml`（path 用原名，无 emoji）
3. 跑 sync → 更新 CSS + 索引文件名 + 链接
4. Obsidian：确认外观已启用 snippet `vault-folder-status-colors`
