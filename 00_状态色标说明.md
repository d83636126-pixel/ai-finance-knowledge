# 状态色标说明（文件夹之上）

> **本页位于各主题文件夹之上**，是全库色标的唯一权威说明。  
> 打开库后先看本页 → 再进 [[00_知识库总导航]] → 再进各文件夹 `00_索引`。  
> 细则执行仍见 [[AI协作记忆系统/AI记录规范#状态颜色标注（2026-07-13 起）]]。

---

## 一、五色含义

| 色标 | 含义 | 何时用 |
|------|------|--------|
| 🟢 | 已完成 / 稳态可参考 | 无未闭环待办，或仅作资料库 |
| 🟡 | 进行中 | 本周正在推进、有未勾选执行项 |
| 🟠 | 日程待办（未到期） | 已排期（如 ~7/31、8/7），尚未开工 |
| 🔴 | 阻塞 / 已逾期 | 过期未做、路径不可用、必须优先解阻 |
| ⚪ | 暂停 / inactive / 暂不做 | 明确挂起，不进本周优先 |

## 二、三层看见方式（必须同时满足）

```text
① 根目录本页（文件夹之上）     ← 你在这里：约定是什么
② 各文件夹 00_索引 标题色标     ← 点进文件夹第一眼
③ 左侧文件列表文件夹着色 CSS   ← 不点开也能扫目录树
```

| 层 | 落点 | 作用 |
|----|------|------|
| 之上 | 本文件 + 总导航置顶 | 说明约定，禁止只写在子文档深处 |
| 文件夹 | 每个 `00_索引.md` 标题前缀色标 + 状态 callout | 进入该区即知忙闲 |
| 列表树 | `.obsidian/snippets/vault-folder-status-colors.css` | 资源管理器里文件夹名带色 |

启用 CSS：Obsidian → 设置 → 外观 → **CSS 代码片段** → 打开 `vault-folder-status-colors`（若未亮，开一次即可；`appearance.json` 已写入启用项）。

## 四、谁负责改色

1. 某文件夹待办清空 → 看板与该夹 `00_索引` 改 🟢，并改 CSS 注释/颜色类。  
2. 新开执行 → 改 🟡。  
3. 排期未做 → 🟠；过期 → 🔴。  
4. 暂停项目 → ⚪。  
5. **先改本页看板，再改文件夹索引，最后改 CSS**（三处一致）。
