## 颜色含义（由状态计算，勿手填颜色）

| 图标 | 侧栏色 | 典型含义 |
|---|---|---|
| 🔴 | 红 | 阻塞，或已超过 `review_by` |
| 🟠 | 橙 | 需要关注、排期中、或等待外部条件 |
| 🔵 | 蓝 | 正在推进（`workflow: active`） |
| 🟢 | 绿 | 已完成 / 稳态可用（`workflow: done`） |
| ⚪ | 灰 | 暂停、归档、或尚未排期 |

两个维度：

- **workflow**：所处阶段（backlog / scheduled / active / waiting / paused / done / archived）
- **attention**：注意程度（normal / watch / blocked；过期由脚本标为 overdue）

例如 `workflow: active` + `attention: blocked` → 仍在推进，但侧栏显示 🔴。

## 怎么查看状态

1. 左侧目录树色条（CSS snippet：`vault-folder-status-colors`）
2. 各夹索引名：`00_<emoji>索引.md`
3. 一页总览：[[00_状态总览]]（立即处理 / 近期关注 / 进行中 / 稳态归档）

**当前文件夹状态表不再人工维护**，以 YAML + 总览为准。

---

## 冲突时谁优先

侧栏只显示一种颜色，优先级从高到低：

1. `attention: blocked` → 红  
2. 超过 `review_by` → 红  
3. `attention: watch` → 橙  
4. `workflow: scheduled` / `waiting` → 橙  
5. `workflow: active` → 蓝  
6. `workflow: done` → 绿  
7. `paused` / `archived` / `backlog` → 灰  

文件级覆盖（`file_status.yml`）只用于与夹色不同的关键文件；普通资料默认继承文件夹色。

## 怎么改状态

1. 改夹级状态 → 编辑 `folder_status.yml`（填 `workflow` / `attention` / `next_action` / `review_by` / `reason`）
2. 个别文件与夹色不同 → 写入 `file_status.yml`（path 用原名，无 emoji）
3. 运行同步：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "F:\AI 金融知识点\90_系统配置\sync_folder_status_colors.ps1" -Check
powershell -NoProfile -ExecutionPolicy Bypass -File "F:\AI 金融知识点\90_系统配置\sync_folder_status_colors.ps1" -Sync -Dashboard
```

4. Obsidian：确认已启用 snippet `vault-folder-status-colors`

**禁止**给普通文件改名为 `🔵_xxx.md`（易坏双链）。只有夹级索引保留 emoji。
